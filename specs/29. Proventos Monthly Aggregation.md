## Overview

Refatora/estende `services/proventos.py` para expor uma função de
**agregação mensal** que o endpoint do chart (Spec 30) consumirá. A
lógica de UNION com OPTION_PREMIUM sintético **já existe** — o foco
aqui é:

1. Adicionar campos derivados que faltam para o chart (`ym`, `klass`,
   `country`, `fi_id`).
2. Implementar agregação por dimensão (klass | country | fi | type | total).
3. Implementar conversão de moeda (BRL ↔ USD) usando `fx_rate` da
   row ou PTAX do mês.
4. Garantir que OPTION_PREMIUM **mantém** a regra de exclusão de DY
   ([[multicurrency_fx_rate_design]]).

Sem novo endpoint aqui — esse vem no Spec 30.

### 3. Conversão de moeda

- **Para BRL**: cada row USD → multiplica `net_amount × fx_rate`
  (PTAX do dia do evento já gravado em `Distribution.fx_rate` ou
  `AssetMovement.fx_rate`).
- **Para USD**: cada row BRL → divide `net_amount / ptax_month(ym)`.
  - `ptax_month(ym)` = PTAX médio do mês (ou último dia útil) via
    `PTAXRate` (Spec 11 já popula). Se o mês não tem PTAX
    (improvável), usar o PTAX mais próximo anterior.

Adicionar helper:


### 5. Filtro de período

```python
def period_range(period: Period, now: date | None = None) -> tuple[date, date]:
    """Returns (from_date, to_date) inclusive."""
```

- `12m`: últimos 12 meses cheios + mês corrente parcial = 13 buckets
- `24m`: análogo
- `ytd`: janeiro do ano corrente até hoje

Decisão: **a row do mês corrente é incluída como bucket parcial**;
o chart mostra normalmente. Frontend pode opcionalmente dim a última
barra (V1.5).

### 6. include_synthetic

Quando `include_synthetic=False`, simplesmente não inclui as rows de
`source=option_premium` retornadas por `list_proventos`. A função
`aggregate_proventos` deve respeitar isso.

Quando `breakdown="type"` e `include_synthetic=False`: legenda **ainda
deve listar** `OPTION_PREMIUM` (com `value=0` em todas as rows) — o
chart precisa disso para o chip dim no card "Por tipo" (specs 32/33).

### 7. Regra de DY (não regredir)

`dy_eligible_amount_brl()` já exclui OPTION_PREMIUM (proventos.py:108).
Este spec **não toca** essa função. Apenas adiciona testes garantindo
que continua exclusiva mesmo após o refactor.

## Out of Scope

- Endpoint HTTP — spec 30
- Caching de `ptax_month` (cache pequeno em memória é OK; cache Redis
  fica pra deploy)
- Drill-down (clicar numa barra → lista filtrada) — Open item
- 36M de período

## Acceptance Criteria

1. `aggregate_proventos` retorna 13 rows para `period="12m"` em um
   dataset com 24m de dados.
2. `currency="USD"` converte rows BRL usando PTAX do mês; total
   bate dentro de ±0,5% comparado a soma manual.
3. `breakdown="type"` retorna 5 keys quando `include_synthetic=True`,
   ainda 5 keys (com OPTION_PREMIUM zerado) quando `False`.
4. `breakdown="klass"` segmenta corretamente; OPTION_PREMIUM rola pra
   klass do underlying (não vira klass "OPTION").
5. `dy_eligible_amount_brl()` continua excluindo OPTION_PREMIUM.
6. `pytest -q` verde (≥8 testes novos).
