# Spec 16 — Sync Numis-Geek → Notion (one-way push)

## Status: 🚧 In progress

## Overview

Empurra entidades do Numis-Geek pras DBs do Notion correspondentes, mantendo o
Notion como espelho read-only. Idempotente via `external_id` =
`notion_page_id`. Detecta edições remotas (você ainda edita no Notion via
mobile) e pede confirmação antes de sobrescrever.

## In Scope

### Backend
- Migration `b7c8d9e0f1g2_notion_sync`:
  - `external_id` + `external_source` em `portfolio_snapshot` e
    `corporate_action` (Asset/AssetMovement já têm)
  - 4 colunas em todos os 4: `notion_last_synced_at`,
    `notion_remote_last_edited_at`, `notion_sync_status`, `notion_sync_error`
- Enum `NotionSyncStatus` (PENDING / SYNCED / CONFLICT / ERROR)
- `src/numis_geek/integrations/notion.py`: httpx wrapper de Notion API
  (create_page, update_page, retrieve_page, query_database).
  Lê token + DB IDs do `IntegrationCredential`.
- `src/numis_geek/services/notion_sync.py`:
  - `push_asset(db, asset)` — DB IG Ativos
  - `push_asset_movement(db, m)` — empurra asset primeiro se PENDING; depois
    cria/atualiza no DB IG Lançamentos
  - `push_snapshot(db, snap)` — cria header em Lote Apuracao + items em Apuracao
  - `push_corporate_action(db, ca)` — DB IG Eventos
  - `detect_conflict(local, remote_last_edited)` — compara
    `notion_remote_last_edited_at` salvo com `last_edited_time` atual da
    page. Se diferente → CONFLICT, não sobrescreve.
  - `resolve_relation_by_title(db_id, title)` — cacheia lookups de Carteira /
    Classe / País / IF (relations no Notion).
- `src/numis_geek/api/routes/notion_sync.py`:
  - `POST /notion-sync/{entity}/{id}` — push 1 + retorna status
  - `POST /notion-sync/{entity}/bulk` — push todos com status PENDING/ERROR
  - `GET /notion-sync/pending` — counts por entidade
  - `POST /notion-sync/{entity}/{id}/resolve` — `{action: "force_push"}` ignora
    `notion_remote_last_edited_at` e sobrescreve
- IntegrationCredential entries: `NOTION/NOTION_TOKEN`,
  `NOTION/DB_IG_ATIVOS`, `NOTION/DB_IG_LANCAMENTO`,
  `NOTION/DB_IG_APURACAO`, `NOTION/DB_IG_LOTE_APURACAO`,
  `NOTION/DB_IG_EVENTOS`.

## Conflict detection

Antes de UPDATE:
1. GET page por `external_id`
2. Lê `last_edited_time` da response
3. Se `last_edited_time` > `notion_remote_last_edited_at` salvo aqui
   E `notion_remote_last_edited_at` não-NULL → CONFLICT
4. UI mostra diff (campos locais vs remotos), 2 botões
5. "Forçar push" → POST /notion-sync/{entity}/{id}/resolve com
   action="force_push" → ignora check, sobrescreve, atualiza
   `notion_remote_last_edited_at` com o novo `last_edited_time`

## Acceptance Criteria

- Migration aplicada limpa
- Botão sync no detalhe do lançamento funciona (cria page no Notion)
- Bulk sync no header processa só PENDING/ERROR
