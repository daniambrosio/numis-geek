## Overview

`/assets/{id}` mostrava um sparkline com dados 100% fictícios —
interpolação `avg_cost → current_price` com seno (`AssetDetail.tsx`
linhas 207-219, copiada literalmente do proto). Spec 46 substitui
isso por **dados reais derivados dos snapshots existentes**:

- ABT (USD): 17 pontos reais
- PETR4 (BRL): tantos quantos snapshots tiver
- Asset novo sem snapshot: card escondido (não simula nada)

V2 com ingest histórico via yfinance/brapi entra como spec própria
depois que confirmarmos rate-limits das APIs.

## In Scope

### 1. Backend — `GET /assets/{id}/price-history`

`src/numis_geek/api/routes/assets.py` ganha:

- Schemas `AssetPriceHistoryPoint` (date YYYY-MM-DD + unit_price str) e
  `AssetPriceHistoryOut` (asset_id, currency, period, points).
- Rota `GET /assets/{asset_id}/price-history?period=6m|12m|24m|all`,
  default `24m`. Mesma auth do `/position`.
- Query: `PortfolioSnapshot JOIN PortfolioSnapshotItem` filtrando por
  `workspace_id`, `asset_id`, `unit_price IS NOT NULL`, e período se
  não for `all`. Ordenado ASC por `period_end_date`.
- 4 testes em `tests/test_assets.py`:
  1. Retorna pontos cronológicos pra workspace user.
  2. Cross-workspace → 404.
  3. period=6m filtra snapshots de 3 anos atrás; period=all inclui.
  4. Asset sem snapshot retorna lista vazia.

### 2. Frontend — chart real + period selector

`frontend/src/lib/api.ts` ganha tipos + helper
`getAssetPriceHistory(id, period)`.

`frontend/src/pages/AssetDetail.tsx`:
- Remove o `priceSeries` simulado.
- `[pricePeriod, setPricePeriod]` state (default `'24m'`).
- `[priceHistory, setPriceHistory]` state — useEffect refetch quando
  id/period mudar.
- Card renderiza só com `priceSeries.length >= 2`.
- SectionTitle ganha GroupingToggle 6M/12M/24M/Tudo + sublabel
  `${N} fechamentos · ${currency}`.
- Título dinâmico: "Preço · 6 meses" / "12 meses" / "24 meses" / "tudo".
- `PriceChartAxis` componente: 4 labels mês/ano evenly-spaced dos
  pontos reais + "hoje" no final.
- `fmtMonthYY(iso)` helper local (pt-BR mês abreviado).

`frontend/src/pages/AssetDetail.test.tsx` (novo) — 3 testes:
1. Renderiza card quando endpoint retorna ≥ 2 pontos.
2. Esconde card quando endpoint retorna < 2 pontos.
3. Esconde card quando endpoint falha.

## Out of Scope

- **Ingest histórico via yfinance/brapi/FRED** — V2 spec própria
  depois de confirmar rate-limits (item 2 de `docs/next-steps.md`).
  Quando vier, ele alimenta nova tabela `asset_price_history` que o
  mesmo endpoint serve, mantendo o contract.

- **Benchmark overlay** (Ibovespa, S&P500) — spec futura.
