# Spec 15 — Dashboard v2

## Status: ✅ Done

## Overview

Liga o dashboard ao patrimônio real: market value (current_price × qty
com PTAX) em vez de só "invested". Adiciona série temporal MoM dos
snapshots quando existirem.

## In Scope

- Header KPIs: patrimônio em **valor de mercado** (BRL) em vez de só
  total investido. Pill "Ganho/perda" mostra diferença. Pill
  "Proventos" destacada.
- Donut por classe / BR vs US / Custodiantes agora usam
  `current_value_brl` (com fallback pra invested quando current_price
  não setado).
- Card "Evolução" carrega snapshots via `api.listSnapshots()` e mostra:
  - Último valor + variação % vs primeiro snapshot
  - Sparkline SVG dos últimos 12 snapshots
  - Empty-state com instrução pra rodar `snapshot create`
- `api.ts` ganha `listSnapshots` / `createSnapshot` + tipo `SnapshotOut`.

## Out of Scope

- Toggle BRL ↔ USD no header (vem com spec multi-moeda futura)
- Drilldowns clicáveis (Patrimônio page)
- Comparação MoM com snapshot anterior na ficha de cada classe

## Acceptance Criteria

- ✅ TS build limpo
- ✅ Donut + HBars usam current_value
- ✅ Sparkline aparece quando há snapshots
- ✅ Empty-state quando 0 snapshots
- ✅ Pytest verde (189 testes — backend não mudou)
