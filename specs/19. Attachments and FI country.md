## Overview

Dois entregáveis num spec só (escopo pequeno):

1. **`FinancialInstitution.country` (ISO-2)** — coluna nova, backfill
   manual das 13 FIs (3 US, resto BR).
2. **`Attachment` polimórfica** — tabela nova com `source_type ∈
   {asset, movement, distribution}` + `source_id`. Storage local em
   `./data/attachments/{workspace_id}/{uuid}.{ext}`. Endpoints CRUD.

Não toca em UI ainda (a UI de notes & attachments inline vem no Spec 21
quando entram `DistributionComposer`/`DetailPanel`).

Bonus: `make backup` (dump SQL + timestamp) pra ter texto-versionável do
estado do DB.

## In Scope

### 1. Migration `f9a0b1c2d3e4_attachments_and_fi_country`

**`financial_institution`** — coluna nova:
- `country` String(2), nullable=False, server_default `'BR'`
  - Antes do `NOT NULL`, fazer `UPDATE` pros 3 FIs US:
    `Avenue`, `Coinbase`, `Wise` → `'US'`.

**`attachment`** — tabela nova:

```sql
CREATE TABLE attachment (
    id            VARCHAR(36) PRIMARY KEY,
    workspace_id  VARCHAR(36) NOT NULL REFERENCES workspace(id),
    source_type   VARCHAR(20) NOT NULL,
        -- 'asset' | 'movement' | 'distribution'
    source_id     VARCHAR(36) NOT NULL,
    kind          VARCHAR(20) NOT NULL,
        -- 'image' | 'pdf' | 'csv' | 'other'
    filename      VARCHAR(255) NOT NULL,
    mime_type     VARCHAR(100) NOT NULL,
    size_bytes    INTEGER NOT NULL,
    storage_key   VARCHAR(500) NOT NULL,
        -- relative path under ./data/attachments/
    uploaded_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    uploaded_by   VARCHAR(36) REFERENCES "user"(id),
    is_active     BOOLEAN NOT NULL DEFAULT 1,
    CHECK (source_type IN ('asset', 'movement', 'distribution'))
);
CREATE INDEX ix_attachment_source ON attachment(source_type, source_id);
CREATE INDEX ix_attachment_workspace ON attachment(workspace_id);
```

Backup do DB antes: `numis_geek.db.bak-before-19`.

### 2. SQLAlchemy model — `models/attachment.py`

```python
class AttachmentSourceType(str, Enum):
    ASSET        = "asset"
    MOVEMENT     = "movement"
    DISTRIBUTION = "distribution"

class AttachmentKind(str, Enum):
    IMAGE = "image"
    PDF   = "pdf"
    CSV   = "csv"
    OTHER = "other"

class Attachment(Base):
    __tablename__ = "attachment"
    id: str
    workspace_id: str FK→workspace
    source_type: AttachmentSourceType
    source_id: str           # FK na tabela apontada por source_type (app-level)
    kind: AttachmentKind
    filename: str            # nome original do arquivo
    mime_type: str           # ex: image/png
    size_bytes: int
    storage_key: str         # path relativo a ./data/attachments
    uploaded_at: datetime
    uploaded_by: str | None FK→user
    is_active: bool          # soft delete
```

### 7. `make backup` target

```makefile
backup:
	@mkdir -p data/backups
	@TS=$$(date +%Y%m%d-%H%M%S); \
		sqlite3 numis_geek.db .dump > data/backups/numis_geek-$$TS.sql; \
		gzip data/backups/numis_geek-$$TS.sql; \
		echo "Backup → data/backups/numis_geek-$$TS.sql.gz"
```

`data/backups/` versionada em git? **Não** — gerados, costumam ser
grandes (centenas de KB compactados). `.gitignore` cobre. O usuário
roda manual quando quiser snapshot textual.

## Acceptance Criteria

1. Migration aplicada limpa em `numis_geek.db`.
2. 13 FIs com `country` populado corretamente:
   - 3 FIs US: Avenue, Coinbase, Wise
   - 10 FIs BR: resto.
3. Endpoint `POST /attachments` aceita PNG/JPEG/WebP/PDF/CSV (5
   tipos), rejeita outros, rejeita > 10 MB.
4. Endpoint rejeita upload pra `source_id` que não existe ou de outro
   workspace (com role admin/member).
5. Sysadmin upload pra qualquer workspace funciona.
6. Files persistem em `./data/attachments/{ws}/{uuid}.ext` no
   filesystem.
7. `make backup` gera `.sql.gz` em `./data/backups/` com timestamp.
8. `npm run build` limpo.
9. pytest verde (incluindo tests novos pra attachments).
