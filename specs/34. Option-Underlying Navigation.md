# Spec 34 — Option ↔ Underlying Navigation

## Status: ✅ Done — 2026-05-24

## Source of truth

- `docs/prototype-deltas-2026-05-23.md` §4.5 "Navegação entre opção e underlying"
- `prototypes/index.html`:
  - `OpenOptionsCard` (~3955) — card no detalhe do underlying
  - Bloco inline `isOption(a) && a.underlying && (...)` em `AtivoDetailPage` (~4180) — fonte do `OptionContextCard`
- Spec 17 ("Opções B3") — backend de OPTION asset class + lifecycle

## Overview

Hoje o usuário consegue ver as opções vendidas sobre um underlying (card
`<OpenOptionsCard />` no `/assets/{itub4_id}`), mas:

1. **Não consegue clicar na opção** — só dá pra mudar de página manualmente.
2. **No detalhe da opção** (`/assets/{itubr364_id}`), não há nenhuma referência
   visual ao underlying — o usuário fica órfão do contexto que justifica
   a operação (cotação atual do ITUB4, distância pro strike, ITM/OTM).

Spec 34 fecha esses dois micro-fluxos. Mexe **só na navegação** —
nenhum dado novo, nenhum schema novo, nenhuma lógica nova de negócio.

## In Scope

### 1. `OpenOptionsCard` — rows clicáveis

Cada opção listada vira navegável para `/assets/{option_id}`:
- Row inteira é o link (não só o ticker).
- Hover: border indigo + chevron-right.
- Botões "Exercer" / "Virou pó" continuam funcionando (preventDefault).
- Mantém os 4 KPIs por opção (Prêmio, Mark-to-market, Se fechasse,
  Preço efetivo).

### 2. `<OptionContextCard />` no detalhe da opção

Novo componente. Render condicional:
`asset.asset_class === 'OPTION' && asset.underlying_id`.

Posição: **logo abaixo do header card**, antes do price chart.

Conteúdo:
- **Header**: ticker do underlying (link pra `/assets/{underlyingId}`),
  cotação atual em destaque, dot de freshness, idade da cotação.
- **4 tiles**:
  - Strike + texto "Comprador exerce vendendo/comprando" (PUT/CALL)
  - Distância ao strike (% verde/vermelho) + badge `ITM`/`OTM`
  - Vence em N dias (âmbar se < 14) + data ISO formatada
  - Cenário provável (Provável exercício / Provável virar pó) +
    preço efetivo se exercida
- **Footer**: lembrete sobre prêmio sintético em `/proventos` + link
  "Ver {underlying.ticker} →".

### 3. Fetch do underlying

`AssetDetail.tsx` faz fetch separado de `api.getAsset(underlying_id)`
quando o asset é OPTION. Sem schema change — `AssetOut` já carrega
`underlying_id`, `option_type`, `strike_price`, `expiration_date`.

### 4. Cálculos puros isolados

Extrair pra `frontend/src/lib/option.ts`:
- `isITM(option_type, strike, current_price): boolean`
- `distanceToStrike(strike, current_price): number` (decimal, ex. 0.085)
- `effectivePrice(option_type, strike, premium_per_share): number`
- `daysToExpiration(expiration_date_iso, today?): number`

Mantém componente fino e permite teste unitário.

### 5. Stack de testes frontend (novo)

Vitest + `@testing-library/react` + jsdom (devDeps). Configura `npm test`
e `npm run test:run`. Primeiros testes:
- `lib/option.test.ts` — 4 cenários ITM/OTM (PUT/CALL × price abaixo/acima
  strike), distância, preço efetivo, vencimento.
- `components/OptionContextCard.test.tsx` — render apenas quando
  OPTION + underlying_id; link aponta pra `/assets/{underlyingId}`.
- `components/OpenOptionsCard.test.tsx` — row renderiza como `<a>` com
  `href="/assets/{option_id}"`; click em "Virou pó" não navega.

## Out of Scope

- Inverter P&L para long options (short-only no V1, conforme prototype).
- Esconder DY/YoC do AssetDetail quando OPTION (ver open items §7 do delta doc).
- Composer dedicado de opção (existe `OptionModal`; sem mudança aqui).
- Mudar schema do `Asset` ou da resposta da API.
- E2E (sem Playwright/Cypress no projeto).

## Acceptance Criteria

1. `/assets/{itub4_id}` mostra `OpenOptionsCard` com as 2 opções; cada
   row é `<a>` com `href="/assets/{option_id}"`.
2. Clicar numa opção navega pro detalhe da opção sem reload.
3. `/assets/{option_id}` renderiza `OptionContextCard` entre header e
   chart, mostrando ticker + preço + freshness do underlying.
4. ITM/OTM corretos:
   - PUT, price < strike → ITM
   - PUT, price > strike → OTM
   - CALL, price > strike → ITM
   - CALL, price < strike → OTM
5. Click em "Virou pó" / "Exercer" dentro de uma row do
   OpenOptionsCard **não** navega pra detalhe da opção (preventDefault).
6. Vitest verde (≥ 3 test files); pytest verde; `npm run build` limpo.
