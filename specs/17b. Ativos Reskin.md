# Spec 17b — Ativos Re-skin

## Status: 🚧 In Progress

## Overview

Second spec of **Fase A** (frontend re-skin against current backend). Reshapes
`/assets` and `/sysadmin/assets` to match the prototype: filter chips replacing
selects, grouping toggle, asset class color dots, currency pills, slide-over
detail panel on row click.

**No backend changes.** Edit / create / deactivate flows keep working via the
existing `AssetModal`.

---

## In Scope

### Visual changes — admin `/assets` table

- **Class dot + label** per row using `KLASS_COLORS` from `lib/tokens.ts`
- **Currency pill** (BRL amber / USD emerald) using `CURRENCY_PILL`
- **`tnum`** on the updated-at column
- **`.money`** class is N/A on this page (no monetary values rendered today)
- Inactive rows keep `opacity-60`; strikethrough on the name when inactive

### Toolbar — filter chips replace selects

- **Search input** stays (full-text on name / ticker / CNPJ)
- **Class filter chips** — horizontal scrollable strip of 14 chips (one per current asset class). Click toggles active. Active = `bg-indigo-50 + indigo-700` (per token), inactive = neutral border. Single-select (matches current `classFilter` shape).
- **Currency chips** — BRL / USD, two small chips with currency pill styling. Single-select.
- **"Mostrar inativos"** stays as a checkbox to the right.
- **Active filter chips become removable** with an `×` button when active.

### Grouping toggle

- Three pills: `Flat` (default) / `Por classe` / `Por custódia`
- When grouping is active, table renders section headers above each group, with item count and a chevron toggle to collapse/expand. Country grouping intentionally absent — depends on `Asset.country` from spec 09.

### Slide-over Asset Detail Panel

- Click a table row → slide-over panels in from the right (`right-0 w-[480px]`)
- Header: large class dot, ticker (mono), name, custodian, class label, currency pill
- KPIs grid (4 cells from `api.getAssetPosition`):
  - Posição (quantity_held)
  - Preço médio (average_cost, in asset currency)
  - Total investido (total_invested_brl)
  - Total recebido (total_received_brl)
  - Preço atual / Variação / Rentabilidade — display **"—"** with subtle "depende de spec 09" tooltip
- Section "Lançamentos recentes" — top 5 from `api.listAssetLancamentos`, asset class colored, click goes nowhere (full table is `/lancamentos`)
- Details: subtype, CNPJ, external_source, created_at, updated_at, notes (textarea, readonly)
- Footer: `Editar` button (opens AssetModal) + `Desativar` button (opens current confirm dialog)
- ESC / outside click closes the panel

### sysadmin `/sysadmin/assets`

Same treatment, plus:

- Workspace selector chip in the toolbar (replaces the dropdown next to the H1)
- Extra column "Workspace" in the table
- Same slide-over panel (no extra fields needed beyond workspace name in header)

---

## Out of Scope

- Country grouping / country column (Asset.country lives in spec 09)
- Numeric KPIs that depend on `current_price` (Variação, Rentabilidade, P&L, current value)
- Asset detail full page (`#/ativo/{id}`) — only slide-over for now
- Inline editing of monetary cells
- Bulk actions (multi-select rows)
- AssetModal redesign — kept unchanged
- Empty / loading state polish beyond what exists

## Migration / Risk

- **Zero data risk.** Pure UI rewrite.
- **Risk of regression on existing flows** (edit, deactivate, search, filter): mitigated by reusing `AssetModal` and the existing API calls verbatim.
- **Behavior change:** filter selects become chips; users who muscle-memory'd the selects need to relearn. Acceptable for a single-owner app being actively rebuilt.
