## Overview

Eleva o `current_price` do AssetDetail a primeira classe visual:
- Botão de refresh (ou edit manual) no header de actions.
- KPI "Preço atual" com dot colorido de freshness e sub-texto
  `há Xh · {source}` (+ conversão BRL se USD).

## In Scope

### 1. Header action

`frontend/src/pages/AssetDetail.tsx` — no header, antes do botão
"Editar":

```tsx
{asset.price_source === "MANUAL" ? (
  <Button icon="edit-2" onClick={openManualPriceModal}>
    Editar preço
  </Button>
) : (
  <Button
    icon="refresh"
    onClick={refresh}
    loading={refreshing}
  >
    Atualizar preço
  </Button>
)}
```

`openManualPriceModal` é wired no Spec 28; aqui só o botão.

### 2. KPI "Preço atual"

No grid de KPIs do AssetDetail, o tile "Preço atual" passa a renderizar:

```
┌─────────────────────────────┐
│ Preço atual           ● tier│  ← dot top-right
│ R$ 39,12                    │
│ há 9h · brapi               │  ← sub-text
│ ≈ US$ 7,85                  │  ← se asset.currency = USD
└─────────────────────────────┘
```

Dot tooltip explica o tier ("Atualizado nas últimas 24h" / "Antigo —
mais de 7 dias", etc.).

### 3. Conversão BRL/USD

Se `asset.currency = USD`: mostra sub-line "≈ R$ X,XX" usando PTAX
mais recente.
Se `asset.currency = BRL`: não mostra conversão.

PTAX vem do mesmo endpoint que já popula o AssetDetail (verificar se
`/portfolio` ou `/assets/:id` retornam — se não, adicionar campo
`fx_rate_today` no asset detail response).

### 4. Estado refresh

Idêntico a Spec 26: estado local `refreshing`, otimista, toast em
erro.

### 5. Loading & erro

- Botão mostra spinner enquanto running.
- Erro: toast "Falha ao atualizar PETR4: timeout" + dot fica `OLD`.
- Sucesso: KPI re-renderiza com novo preço + idade "agora".

## Out of Scope

- Modal de edição manual em si — spec 28
- Histórico de preços / sparkline no KPI — spec futuro
- Adicionar `price_source` editável no header (read-only por enquanto)

## Acceptance Criteria

1. AssetDetail mostra botão "Atualizar preço" (refresh icon) quando
   automatizado.
2. AssetDetail mostra botão "Editar preço" (edit icon) quando MANUAL
   (handler stub OK até spec 28).
3. KPI "Preço atual" tem dot colorido no canto superior direito
   correspondente ao `price_tier`.
4. Sub-text mostra `há Xh · {source label}`.
5. Asset USD mostra `≈ R$ X,XX` em linha extra.
6. Click no refresh atualiza preço + idade in-place sem reload da
   página.
7. Build limpo.
