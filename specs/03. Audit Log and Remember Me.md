# 03. Audit Log and Remember Me

## Purpose
Track all meaningful actions in the system for accountability and debugging. Also improve the login experience with a "remember me" option that controls token lifetime and storage.

---

## A. Remember Me

### Backend
`POST /auth/login` accepts `remember_me: bool = False`.
- `False` → JWT TTL = 8 hours
- `True` → JWT TTL = 30 days

### Frontend
- Checkbox "Lembrar de mim" on login form
- `remember_me=False` → token stored in `sessionStorage` (cleared when browser closes)
- `remember_me=True` → token stored in `localStorage` (persists 30 days)
- `api.ts` reads token from `localStorage` first, then `sessionStorage`
- Logout clears both storages
- `PrivateRoute` checks both storages

## B. Audit Log

### Model: `audit_log`

| Column | Type | Notes |
|---|---|---|
| id | UUID | PK |
| workspace_id | str \| None | nullable for failed/anonymous actions |
| user_id | str \| None | nullable |
| user_email | str | snapshot at time of action |
| action | str | dot-notation key (see table below) |
| resource_type | str \| None | e.g. `user`, `workspace` |
| resource_id | str \| None | ID of affected resource |
| details | str \| None | JSON string with extra context |
| created_at | datetime | UTC, auto |

### Explicit action keys

| Route | Action |
|---|---|
| `POST /auth/login` (success) | `auth.login` |
| `POST /users/invite` | `user.invited` |
| `PUT /users/{id}/role` | `user.role_changed` |
| `PUT /users/{id}/deactivate` | `user.deactivated` |
| `PUT /users/me` | `profile.name_changed` |
| `PUT /users/me/password` | `profile.password_changed` |

### Automatic middleware
FastAPI middleware logs all POST/PUT/DELETE requests that return 2xx with action key `http.{method}.{path}`. Serves as safety net for future routes without explicit logging.

### API endpoint
`GET /audit` — admin only
- Query params: `page=1`, `limit=50`, `action=` (optional filter)
- Response: `{ items: AuditLogOut[], total: int, page: int, pages: int }`
