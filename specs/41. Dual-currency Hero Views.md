## Overview

Hoje o app mostra valores quase só em BRL. Quem opera com exposição USD
relevante (ETFs US, Wise, cripto) tem que **fazer conta de cabeça** pra
ter ideia de patrimônio em moeda forte ou comparar com benchmarks
internacionais.

Esta spec entrega a versão **mínima** do dual-currency: em 3 telas-chave
(Dashboard hero, Patrimônio hero+tabela, Snapshot detail+lista), cada
valor monetário agregado mostra **BRL como principal e USD em dim
embaixo** — sem toggle, sem ambiguidade, sem refator massivo.

**Filosofia**: zero estado, zero preferência salva. O usuário sempre vê
os dois. Quem só pensa em BRL ignora a sub-linha; quem quer pensar em
USD olha um pixel pra baixo. Tela densa o suficiente pra caber, leve
o suficiente pra não poluir.

**Out of scope expressamente**: toggle global, conversão em tabelas
densas (Lançamentos, Proventos, AssetMovements), conversão em fields
de input ou detail panels.

## Decisão arquitetural prévia

Discutido na sessão e descartado:
1. **Toggle global** (Spec 39+40 originais): custo alto (refator de 50+
   chamadas de `fmtBRL`/`fmtUSD`, estado novo, persistência, modo
   privacy interaction). Pay-off provavelmente baixo até o user provar
   uso frequente. Reabrir como Spec 42+ quando virar pedido.
2. **Per-tela toggle**: fragmentação de estado, pior UX.
3. **Settings modal**: muito escondido pro caso de uso real.

Vencedor: **persistent dual display** nas 3 telas hero. Cobre 70% do
valor por 20% do esforço.

## In Scope

### 1. Dashboard hero (`pages/Dashboard.tsx`)

Os **3 tiles secundários** ("Investimentos", "Caixa", "Cartões abertos"
no protótipo Dashboard ~2575) ganham sub-linha USD em dim
(`text-gray-500 dark:text-gray-600`, font-size 10px, mesma indentação).

```tsx
<div className="px-3 py-2 rounded-lg bg-gray-100 dark:bg-gray-800/50">
  <div className="text-[10px] uppercase tracking-wider text-gray-500">Investimentos</div>
  <div className="text-sm font-semibold tnum money text-emerald-400">+ {fmtBRL(totalInvest, { compact: true })}</div>
  <div className="text-[10px] tnum money text-gray-500 dark:text-gray-600 mt-0.5">+ {fmtUSD(totalInvest / ptaxNow, { compact: true })}</div>
</div>
```

Conversão usa **PTAX do dia** (`ptaxNow`) — não fx_rate de evento, porque
o tile é "agora". Backend: estender `GET /portfolio` para retornar
`investments_brl`, `investments_usd`, `cash_brl`, `cash_usd`,
`cards_open_brl`, `cards_open_usd` (ou passar `ptax_now` e converter
no front).

**Hero big number** (R$ 2.834.000 + $565.234 abaixo): já mostra os
dois hoje. Trabalho aqui é remover o `<CcyPill ccy="BRL"/>` ao lado e
o `<CcyPill ccy="USD"/>` da sub-linha (regra §3).

### 2. Patrimônio hero + Top 10 (`pages/Patrimonio.tsx`)

Mesma alteração do Dashboard nos 3 tiles secundários.

Tabela "Top 10 posições" — coluna **Valor**:
- Cabeçalho hoje: `"Valor (BRL)"`. Vira `"Valor"`.
- Cada célula vira 2 linhas:

```tsx
<td className="px-2 text-right">
  <div className="tnum money font-medium">{fmtBRL(asset.valueBRL, { compact: true })}</div>
  <div className="tnum money text-[10px] text-gray-500 dark:text-gray-600">{fmtUSD(asset.valueBRL / ptaxNow, { compact: true })}</div>
</td>
```

### 3. Snapshot detail (`pages/SnapshotDetail.tsx`)

Aqui o pay-off é maior — snapshot já tem `total_value_usd` e
`fx_rate_usd_brl` carimbados na época (Spec 14). **Custo zero** no
backend; só consumir.

KPIs:
- **Patrimônio fim do mês** — já tem `sub` com USD (proto ~7589). OK.
- **Variação MoM** — sub vira `${ΔR$} · ${Δ$}` em vez de `vs R$ X`.
  Cálculo `Δ$ = ΔR$ / snap.fx_rate` (PTAX da época do snap atual).
- **Proventos recebidos** — sub ganha USD: `${fmtUSD(prov_brl / snap.fx_rate)} · N eventos`.
- **Yield on portfolio** — % invariante, sem mudança.

Tabela do histórico (FechamentosPage ~7312) — coluna **Patrimônio**
vira célula 2 linhas com BRL + USD dim. Conversão usa
`snap.total_usd` (já existe) — não recalcular com PTAX de hoje.

### 4. Regra "nasce sem pill" (importante)

`<CcyPill>` continua existindo, mas:

- **Remover** das hero number (Dashboard, Patrimônio) — o stacking BRL/USD
  já é o disambiguador.
- **Manter** em detail panels (Lançamento, Provento, AssetDetail) onde
  o valor flutua isolado sem coluna explicando.
- **Cleanup oportunista das tabelas Lançamentos/Proventos** fica fora
  desta spec — vira Spec 42 quando puxado.

Documentar em `docs/design-system.md` (ou criar): "use `<CcyPill>` apenas
quando o valor está decontextualizado — sem prefixo `R$`/`$` visível,
sem par converso visível, sem coluna explicando."

### 5. Sub-linha — pattern document

Pra próximas telas que adotarem, padronizar:

```tsx
// classes canonicas do dual-display:
const dualMainCls = "tnum money font-medium";              // ou font-semibold em hero
const dualSubCls  = "tnum money text-[10px] text-gray-500 dark:text-gray-600 mt-0.5";
```

Sem componente novo (não vale a complexidade). Se virar pattern em > 5
lugares depois da Spec 41, refatorar pra `<DualMoney brl={} usd={}/>`.

### 6. Conversão de "agora" vs "época"

- **Dashboard + Patrimônio** (estado atual) → `ptax_now` (PTAX do
  último dia útil disponível, vindo do endpoint `/portfolio` ou
  `/ptax/latest`).
- **Snapshot detail + Snapshot list** (estado histórico) → `snap.fx_rate`
  carimbado no snapshot. Importante: **não usar PTAX de hoje** pra
  snapshots históricos — seria revisionismo de valor.

## Out of Scope

- **Toggle global** BRL↔USD. Reabrir como Spec maior se virar pedido
  recorrente.
- **Conversão em tabelas densas** (Lançamentos, Proventos, AssetMovements,
  CreditCardInvoices) — cada linha já mostra ccy original, dual ali
  poluiria 30+ rows por viewport.
- **Conversão em fields de input** (composers) — strike de PUT em BRL
  não vira "$X" mesmo no input.
- **Conversão em detail panels** — valores ficam decontextualizados;
  manter `<CcyPill>` da ccy original.
- **CcyPill cleanup** em tabelas existentes (Lançamentos NET column,
  Proventos line items) — vira Spec 42.
- **PrivacyToggle interaction** — privacy já mascara antes de qualquer
  format; sem mudança.
