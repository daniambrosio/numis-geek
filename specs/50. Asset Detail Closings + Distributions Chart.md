# Spec 50 — Asset Detail · Fechamentos + Proventos bimoeda + gráficos

## Status: 📝 Implementing

## Why

A página `/assets/{id}` mostra o estado atual + movimentações + tabela de
proventos. Faltam duas visões temporais que o user precisa:

1. **Histórico de fechamentos por ativo** — quanto o ativo valeu no
   final de cada mês, nas duas moedas, e como o valor mudou com aportes.
2. **Proventos bimoeda + gráfico** — tabela existente é só BRL e não
   tem visão temporal/sazonal.

## Decisões locked

| # | Pergunta | Resposta |
|---|---|---|
| 1 | Tabela de fechamentos | Período, Qtd, Preço fim, Valor BRL, Valor USD, **Δ MoM (%)**. Cronológica desc. |
| 2 | Gráfico fechamentos | **Linha do valor mensal (BRL) + barra de aporte sobreposta** no mês que teve compra. |
| 3 | Proventos bimoeda | **Coluna USD ao lado da BRL** na tabela existente. Total geral em ambas. |
| 4 | Gráfico proventos | **Bar chart mensal BRL últimos 24m + linha de média móvel 12m**. USD vai no tooltip. |

## In Scope

### 1. Backend

**1.1 `GET /assets/{asset_id}/snapshot-history`**

Retorna lista de items pra cada CLOSED snapshot do workspace que tem
esse ativo:

```python
class AssetSnapshotHistoryItem(BaseModel):
    period_end_date: str          # "2026-04-30"
    quantity: str
    unit_price: str | None
    market_value_brl: str
    market_value_usd: str | None
    fx_rate_usd_brl: str | None   # PTAX usado no snapshot

class AssetSnapshotHistoryOut(BaseModel):
    items: list[AssetSnapshotHistoryItem]   # cronológica desc
```

Filtra `snapshot.status == CLOSED` (não inclui IN_REVIEW pra evitar
poluir o histórico do ativo com dados não confirmados).

**1.2 Endpoint de aportes mensais — REUSO**

O endpoint `GET /assets/{id}/movements` já existe e devolve todos os
AssetMovements. FE agrega por mês client-side (filtra `type=BUY`,
groupBy ymString(event_date), soma `net_amount`). Sem endpoint novo.

### 2. Frontend

**2.1 `<AssetSnapshotsCard />`** — card novo no `AssetDetail.tsx`,
entre o card de KPIs e a seção de Movimentos. Renderiza:

- **Tabela** (linha = 1 snapshot, ordem cronológica desc):

  ```
  Período   Qtd       Preço fim   Valor BRL   Valor USD   Δ MoM
  Mai/26    47.670    R$ 1,71     81.912,21   14.732,56   +2,3%
  Abr/26    47.670    R$ 1,68     80.094,11   15.412,33   -1,5%
  ...
  ```

  - Linhas clicáveis abrem `/snapshots/{ym}` com state `from`
    apontando de volta (back nav consistente com Spec 49 hotfix).
  - Δ MoM colorido (emerald > 0 / red < 0).
  - Quantidades formatadas pt-BR com 4 decimais.

- **Gráfico** (acima ou ao lado da tabela):

  - SVG mini-chart sem dep externa (segue padrão Sparkline /
    SnapshotEvolutionChart existentes).
  - **Linha** do valor mensal BRL (últimos 24 meses, ou tudo que tiver).
  - **Barras finas** sobrepostas no eixo X nos meses com aporte
    (somatório dos BUYs do mês). Cor distinta (indigo).
  - Tooltip ao hover: período, valor, aporte (se houver).

**2.2 Tabela de Proventos bimoeda**

Componente atual em `AssetDetail.tsx` (procurar pelas distribuições
linkadas ao asset_id):

- Adicionar coluna **"Valor USD"** ao lado de "Valor BRL".
- Cálculo: `usd = brl / fx_rate` quando `distribution.currency = BRL`
  e `fx_rate` presente; caso contrário usa `gross_amount_usd`
  persistido (ou `amount_native` se já em USD).
- Linha de total no rodapé nas 2 moedas.

**2.3 `<AssetDistributionsChart />`** — componente novo.

- Bar chart mensal últimos 24 meses (BRL), agrupado por
  `event_date.slice(0,7)`.
- Linha sobreposta da **média móvel 12m** (calculada client-side).
- Tooltip por mês: BRL + USD + breakdown por tipo (DIV/JCP/COUPON)
  num popover compacto.
- Eixo Y BRL principal; USD aparece só no tooltip pra não complicar
  visual.

### 3. Tests

Backend:
- `test_asset_snapshot_history_endpoint` — cria 3 snapshots CLOSED,
  GET retorna 3 items na ordem desc.
- `test_asset_snapshot_history_excludes_in_review` — snapshot
  IN_REVIEW não aparece.

Frontend:
- `AssetSnapshotsCard.test.tsx`: render da tabela com mock + cálculo
  do ΔMoM em verde/vermelho.
- (Gráficos são SVG difíceis de testar com snapshot — render smoke
  test confirma existência de elementos chave).

## Out of Scope

- **Edição inline** de quantidade/preço na tabela de fechamentos
  (já existe a página do snapshot pra isso).
- **Export CSV** dos fechamentos (Spec futuro).
- **Comparação com benchmark** (CDI, IBOV) no gráfico —
  feature dedicada futura.

## Acceptance Criteria

1. `/assets/{id}` mostra a tabela "Fechamentos" com colunas certas.
2. Linhas da tabela com `Δ MoM` colorido.
3. Gráfico abaixo da tabela tem linha de valor + barras nos meses
   de aporte.
4. Tabela de proventos tem coluna USD; total geral em ambas.
5. Gráfico de proventos é bar mensal + linha m12.
6. `pytest -q` + `npm run test:run` + `npm run build` verdes.
