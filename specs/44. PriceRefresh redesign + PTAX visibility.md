## Overview

Spec 41 fez a carteira dolarizada visível em hero numbers e introduziu
dependência forte na PTAX fresh do dia. A topbar tinha um botão
"refresh" (`frontend/src/components/PriceRefresh.tsx`) que (a) divergia
do protótipo em layout/cor de dots, e (b) **não expunha nada de PTAX**
— se o cron de 20h falhasse, ninguém percebia até abrir DevTools.

Esta spec realinha o widget ao protótipo + adiciona um bloco PTAX
in-widget com botão de force-sync acessível a qualquer user
workspace.

## In Scope

### 1. Backend — rotas PTAX workspace-level

`src/numis_geek/api/routes/ptax.py` ganha um `workspace_router` com
prefix `/ptax` (paralelo ao `router` admin `/sysadmin/ptax`):

- `GET /ptax/status` → `PTAXStatusOut` (total_rows, last_date,
  oldest_date, last_fetched_at). Sem `_require_sysadmin`.
- `POST /ptax/sync` body `{mode: 'incremental'|'full'}` →
  `PTAXSyncResultOut`. Reusa `services/ptax_sync.sync_ptax`. Sem
  `_require_sysadmin`.

`src/numis_geek/api/app.py` registra
`app.include_router(ptax.workspace_router)`.

### 2. Frontend — PriceRefresh redesign

`frontend/src/components/PriceRefresh.tsx` reescrito pra seguir o proto:

- **Width**: `w-[360px]` (antes 80).
- **Header**: título "Preços dos ativos" + sub-linha "Mais antigo:
  `<age tier-colored>` · `N ativos`" + botão "Atualizar agora" h-7
  inline top-right.
- **Fontes**: ordem fixa BRAPI → FINNHUB → COINBASE → TESOURO. Cada
  linha: dot 1.5px **cinza-400 default** (indigo-pulse during
  refresh; emerald após sucesso; amber após parcial), label do proto
  ("B3 / brapi", "Finnhub", "Coinbase", "Tesouro Direto"), "· N
  ativos" inline, counter "X/Y" à direita pós-refresh. Linha
  inteira clicável (capability extension preservada).
- **Manual row**: border-t dashed, dot cinza, "Manual · N ativos
  (atualizar no detalhe)", não clicável.
- **Result panel**: emerald bg, check ou alert icon, "X de Y
  atualizados", erros com ticker mono + reason cinza.
- **Mais desatualizados**: título com `(N)` count em parens;
  filtra a **stale-only** (tier ≠ FRESH); each row left-bar 1×4
  amber/red; "Ver todos N → /assets" se > 5; max-h-[200px] overflow.
- **PTAX block** (NEW): título "PTAX" + sub-linha "Último: `<dd/mm/yyyy>`
  · `<age tier-colored>`" + "`N` cotações armazenadas" + botão
  "Sincronizar" h-7 inline. Lazy-fetch quando widget monta.
- **Footer**: "Atualizações automáticas diárias às 18h (preços) e
  20h (PTAX). Ativos manuais (imóveis, veículo) precisam ser
  editados no detalhe."

### 3. Trigger pill age — worst-of

`lib/price.ts` ganha:

- `ptaxTier(lastDateIso, now)` → `PriceTier`. Thresholds: ≤24h FRESH,
  ≤72h STALE, > 72h OLD.
- `worstOfTiers(a, b)` → `PriceTier`. UNKNOWN é tratado como
  no-signal (não rebaixa um tier real).

O widget calcula `pillTier = worstOfTiers(stats.worstTier, ptaxTier(ptax.last_date))`
e usa `pillAge = max(asset oldest age, PTAX age)` no rótulo da pill.

### 4. API client

`frontend/src/lib/api.ts` ganha:

- `ptaxStatusWorkspace: () => request<PTAXStatusOut>('/ptax/status')`
- `syncPtaxWorkspace: (mode) => request<PTAXSyncResultOut>('/ptax/sync', { method: 'POST', body: {mode} })`

(`ptaxStatus` + `syncPtax` admin-only ficam pra page `/sysadmin/ptax`.)

## Out of Scope

- **Refator dos endpoints admin** `/sysadmin/ptax/*` — continuam.
- **Background polling do PTAX** — fetch é lazy quando widget monta.
- **Histograma / sparkline de PTAX no widget** — overkill; vai pra
  page `/sysadmin/ptax` existente.
- **PTAX EUR / outras moedas** — Spec 11 já marcou USD-only V1.
