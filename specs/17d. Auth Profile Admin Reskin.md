# Spec 17d — Auth / Profile / Admin Re-skin Sweep

## Status: 🚧 In Progress

## Overview

Last spec of **Fase A** — sweeps the remaining existing pages
(Login, Profile, Contas, Admin Users, Admin Audit, Sysadmin FIs) so
every page in the app uses the prototype's visual vocabulary before
we kick off Fase B (backend refactor).

**No backend changes.**

---

## In Scope

### Login `/login` — full rewrite

Matches the prototype's LoginPage:
- Gradient backdrop (radial indigo + violet blobs)
- Wordmark on top (`N` gradient square + "Numis-Geek" + "finanças pessoais" caption)
- `Card` with `Entrar` heading + helper text
- Email + Senha fields styled as `Field` (label + input + optional hint)
- Eye toggle inside password input to show/hide
- `ToggleSwitch` "Manter conectado por 30 dias" (replaces checkbox)
- Submit button: indigo when valid, gray-disabled when invalid; arrow icon
- Inline error below form
- Footer caption `v1 · sem signup (single-owner)`

### Profile `/profile` — polish

- Use `Card` + `SectionTitle` primitives
- Move role label into a small badge
- Tighten typography to match `text-[12px]` / `tnum` palette
- Replace 3 stacked sections with one Card "Conta" (read-only) + one Card "Atualizar nome" + one Card "Alterar senha"
- Inputs use the standard rounded-lg + h-8 height

### Contas `/accounts` — rewrite

Match the prototype's `ContasList` layout (current entity model — no Cards entity yet so we skip the "Cartões de crédito →" link):
- `PageHeader` with `Contas`, subtitle `N contas · X tipos · Y instituições`
- Two sections grouped by `account_type` (investment first, then checking)
- Each section: `Card` with `SectionTitle` + grid of `AccountRow`
- `AccountRow`: FILogo + name + type pill (Conta Investimento violet / Conta Corrente blue) + FI short_name + currency + ativos count (for investment) + balance KPI on right
- Action button "+ Nova Conta" stays (keeps current AccountModal)

Asset count per investment account: computed client-side by counting assets where `financial_institution_id === account.financial_institution_id` AND `account.account_type === investment` — approximation until `Asset.account_id` lands in spec 10.

Balance: shows `opening_balance` or "—" since Transactions don't exist yet (spec 11). The slot is there for when balance gets real.

### Users `/admin/users` — polish

- `PageHeader` with `Usuários`, count
- `Card` wrapping the existing table
- Use `text-[12px]` density and the same role badge pattern as Profile

### Audit `/admin/audit` — polish

- `PageHeader` with `Auditoria`, count
- `Card` wrap the table
- Action filter chips replace the existing select
- Compact rows: timestamp (mono tnum), user (small), action (badge-styled), resource type, details
- Pagination buttons kept

### Sysadmin FIs `/sysadmin/financial-institutions` — polish

- `PageHeader` with `Instituições Financeiras`, count
- `Card` wrap the existing table
- FILogo column (instead of/in addition to the colored initials chip)
- Modal kept unchanged

## End of Fase A

After 17d merges, every page in the app uses the prototype's visual
vocabulary. The next round — Fase B, starting at spec 08 — refactors
the backend (rename Lancamento → AssetMovement, Distribution split,
Asset.account_id FK, etc.) which then unblocks the placeholder cards
on Dashboard (BR vs US, Top movers, real Net Worth equation) and the
new pages (Patrimônio, Proventos, Movimentações, Cartões, Faturas,
Orçamento, FI Hub, Asset detail page).
