# Spec 17 — Opções B3 (calls/puts)

## Status: 🚧 In progress

## Source of truth

`docs/options-rationale.md` (aprovado 2026-05-06). Esta spec é o
*implementation contract*; qualquer divergência com o rationale = bug.
Prototype: `prototypes/index.html` (linhas 225-237, 292-294, 3159-3270).

## Overview

Adiciona `AssetClass.OPTION` (12ª classe) com FK pro underlying. Suporta
covered call e cash-secured put (operações de SELL_OPEN), exercício
automático no underlying com preço efetivo (strike ± prêmio/ação), e
prêmios como `OPTION_PREMIUM` (provento sintético via view/UNION — NÃO
cria Distribution row).

## In Scope (V1)

### Migration `d2e3f4a5b6c7_options`

**`Asset`** — colunas novas (nullable; obrigatórias só pra OPTION):
- `underlying_id` String(36) FK→asset.id
- `option_type` Enum(CALL, PUT)
- `strike_price` Numeric(18, 8)
- `expiration_date` Date
- `contract_size` Integer (default 100 pra OPTION)

**Enum `AssetClass`** ganha `OPTION`.

**Enum `AssetMovementType`** ganha:
- `SELL_OPEN` — vender pra abrir (recebe prêmio)
- `BUY_TO_OPEN` — comprar pra abrir (paga prêmio)
- `BUY_TO_CLOSE` — recomprar pra fechar short
- `SELL_TO_CLOSE` — revender pra fechar long
- `EXERCISED` — exercida (BR; cobre US "assigned" também)
- `EXPIRED` — expirou worthless

**`AssetMovement`** — coluna nova:
- `related_movement_id` String(36) FK→asset_movement.id, nullable
  (self-FK; liga BUY/SELL no underlying ao EXERCISED da opção)

> **OPTION_PREMIUM NÃO é DistributionType.** Não cria Distribution rows;
> é só uma view/query union sobre AssetMovement onde type IN
> ('SELL_OPEN', 'BUY_TO_CLOSE').

### `services/positions.py`
Tratamento dos novos types:
- `SELL_OPEN`/`BUY_TO_OPEN`: + qty na opção; net_amount = ± prêmio
- `BUY_TO_CLOSE`/`SELL_TO_CLOSE`: − qty; net_amount = ± valor fechamento
- `EXERCISED`/`EXPIRED`: qty da opção → 0; sem efeito direto no underlying
  (o impacto vem do BUY/SELL related)

### `services/option_lifecycle.py` (novo)
```python
def compute_open_options(db, underlying_id) -> list[dict]:
    """Lista opções ativas sobre o underlying com KPIs: prêmio recebido,
    mark-to-market, close-now P&L, preço efetivo se exercida, dias pra
    vencer, verdict ITM/OTM."""

def exercise_option(db, option_id, exercise_date) -> AssetMovement:
    """PUT vendida → BUY no underlying com price = strike - prêmio/ação.
    CALL vendida → SELL no underlying com price = strike + prêmio/ação.
    Cria EXERCISED no asset opção + BUY/SELL related no underlying.
    Marca option.is_active=False."""

def expire_option(db, option_id, exercise_date) -> AssetMovement:
    """Cria EXPIRED no asset opção. Marca is_active=False. Prêmio fica."""

def parse_br_option_ticker(ticker, underlying_price) -> dict:
    """ITUBR364 → {prefix:'ITUB', month:'Jun', option_type:'PUT',
    strike_raw:'364', strike_suggested:36.40, adjustment_suffix:None}.
    Strike é sugestão — UI confirma."""
```

### `services/proventos.py` (novo helper)
```python
def list_proventos(db, workspace_id, include_synthetic=True) -> list:
    """UNION entre Distribution + AssetMovement(SELL_OPEN, BUY_TO_CLOSE).
    Synthetic rows marcadas type='OPTION_PREMIUM', asset_id = underlying.
    NÃO entram em DY/YoC queries."""
```

### API routes — `api/routes/options.py` (novo)
- `POST /options` — cria OPTION asset + primeiro movement (SELL_OPEN ou BUY_TO_OPEN)
- `GET /options/parse?ticker=ITUBR364` — retorna parser dict (sugestão de strike, type, mês)
- `GET /assets/{underlying_id}/open-options` — opções abertas sobre o underlying com KPIs
- `POST /options/{id}/exercise` — força exercício
- `POST /options/{id}/expire` — marca expired
- `POST /options/{id}/close` — close manual (BUY_TO_CLOSE ou SELL_TO_CLOSE)

### API routes — extensões
- `GET /distributions` ganha `include_synthetic=True` query param que
  faz UNION com OPTION_PREMIUM rows
- `GET /assets` ganha `include_options=False` default (esconde OPTION)
- `Asset.dy_12m` / `yoc_12m` calculations exclude OPTION_PREMIUM

### Frontend

- **`OptionModal`** — composer dedicado:
  - Step 1: cola ticker → parser sugere underlying + type + month + strike
  - Step 2: confirma underlying picker (autocomplete), strike, qty,
    prêmio/ação, tipo da op (default SELL_OPEN)
  - Naked option warning se cobertura insuficiente
- **`OpenOptionsCard`** — embedded em `AssetDetailPanel` quando underlying
  tem opções abertas. Lista cada com:
  - PUT/CALL badge, ticker, strike, dias pra vencer
  - Prêmio recebido (sign), mark-to-market, "se fechasse agora", preço efetivo
  - Verdict "Provável exercício" (ITM) vs "Provável virar pó" (OTM)
  - Ações: Fechar / Exercer / Marcar expired
- **`OptionDetailPanel`** — quando clicar numa OPTION
- `/assets`: toggle "Incluir opções" (default off)
- `/proventos`: toggle "Incluir prêmios sintéticos" (default on);
  categoria OPTION_PREMIUM em violet (#a855f7)
- Sidebar widget Dashboard: "Opções vencendo em 7 dias"
