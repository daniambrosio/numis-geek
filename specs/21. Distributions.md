## Overview

Página de proventos com:
1. **Hero KPIs** — total recebido líquido, bruto, IR retido.
2. **Por tipo card** — 4 tiles (Dividendo / Juros / JCP / Aluguel) com
   total + %.
3. **Filtros** — search (ticker/nome/IF), multi-chip de tipo, toggle de
   visão (mês × ativo).
4. **Tabela** — vista mensal (agrupada por YYYY-MM) ou por ativo.
5. **DistributionComposer** — 4 tipos em grid 2×2; asset opcional para
   `SECURITIES_LENDING` (Avenue manda genérico).
6. **DistributionDetailPanel** — slide-over com fields grid + notes.
7. **Notion import** — 1715 proventos já importados via
   `scripts/import_notion_distributions.py`. Re-run idempotente.

Backend já tinha `Distribution` model + 5 endpoints CRUD (Spec 08); este
spec só adiciona UI.

## In Scope

### 1. `DistributionComposer.tsx` (novo)

Espelha o protótipo:
- Header "Novo Provento / Rendimento, dividendo, JCP ou aluguel".
- Grid 2×2 de cards type-picker (DIVIDEND / INTEREST / JCP /
  SECURITIES_LENDING) com label + hint.
- Date + Instituição (select).
- Asset (select); opcional quando type=SECURITIES_LENDING.
  Filtrado pelos assets do FI selecionado.
- Bruto + IR retido (currency derives from asset.currency).
- Pré-visualização ao vivo (líquido + ativo/origem).
- Notas textarea.
- ⌘↵ salvar / esc fechar.

### 2. `DistributionDetailPanel.tsx` (novo)

Slide-over right-side igual ao `LancamentoDetailPanel`:
- Header com type badge, ccy pill, asset (ou "Sem ticker via {FI}").
- Asset/FI summary card.
- Fields grid: Data, Moeda, Bruto, IR retido, Líquido, FX rate, Origem, Status.
- Notas.
- Footer: Editar / Desativar.

(Sem Notion sync section por enquanto — `push_distribution` não existe
no backend ainda; será adicionado quando o sync bidirecional de
distributions fizer sentido.)

### 3. `Distributions.tsx` (novo, substitui ComingSoon)

- Top: PageHeader com contador + botão "+ Novo Provento".
- 2 cards lado a lado: Total líquido + Por tipo (4 tiles).
- Card de filtros: search + view toggle (Por mês / Por ativo) +
  multi-chip de tipo.
- View=month: tabela agrupada YYYY-MM, com sticky row de cabeçalho de
  mês; click numa linha abre `DistributionDetailPanel`.
- View=asset: agregação por ativo (eventos + total BRL), click navega
  pra `/assets/:id` quando há ticker.

### 4. Wire-up

## Out of Scope

- **OPTION_PREMIUM** synthetic UNION em `/distributions` (toggle "Incluir
  dividendos sintéticos") — fica pra spec quando o backend tiver o helper
  `services/proventos.list_proventos(include_synthetic)` (foi mencionado
  em Spec 17 mas não implementado).
- **Push Notion** das Distributions criadas localmente — sem caso de uso
  v1 (importer é one-way).
- **Filtros por data range** (DateRangeButton no protótipo é só visual).
- **Attachments inline** nos painéis — endpoints já existem (Spec 19),
  UI inline entra em spec dedicado.
