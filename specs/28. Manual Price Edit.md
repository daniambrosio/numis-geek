# Spec 28 — Manual Price Edit (PATCH)

## Status: ✅ Done — 2026-05-24

## Source of truth

- `docs/prototype-deltas-2026-05-23.md` §1.3 (PATCH endpoint)
- Spec 22 (`PriceSource.MANUAL`)
- Spec 27 (botão "Editar preço" no AssetDetail)

## Overview

Permite editar manualmente o `current_price` de ativos com fonte
`MANUAL` — imóvel, veículo, FGTS, VGBL, CDB, LCI, etc. Não há fonte
automática pra esses; o user atualiza ao seu critério (e.g. quando
recebe extrato anual).

## In Scope

### 1. Endpoint

```text
PATCH /api/assets/{id}/price
  body: { "price": "850000.00" }
  rules:
    - asset.price_source DEVE ser MANUAL → 422 caso contrário
    - price >= 0
    - atualiza price + price_updated_at = now()
  resp 200:
    {
      "price": "850000.00",
      "price_updated_at": "2026-05-24T10:23:00-03:00",
      "price_source": "MANUAL"
    }
```

Auth: `member`+ no workspace do asset; sysadmin cross-workspace OK.

### 2. Audit

```
action: "price.update.manual"
resource: "asset/{id}"
details: "Casa Curitiba: R$ 820.000 → R$ 850.000 (MANUAL)"
```

### 3. UI — Modal `<ManualPriceModal />`

Disparado pelo botão "Editar preço" do AssetDetail (spec 27):

```
┌──────────────────────────────────────┐
│ Editar preço · Casa Curitiba         │
│                                      │
│ Preço atual:  R$ 820.000,00          │
│               atualizado há 47 d     │
│                                      │
│ Novo preço:                          │
│ [ R$  850.000,00 ___________ ]       │
│                                      │
│ Notas (opcional):                    │
│ [ Avaliação anual feita pelo... ]    │
│                                      │
│       [ Cancelar ]  [ Salvar ]       │
└──────────────────────────────────────┘
```

- Currency input respeita `asset.currency` (R$ ou US$).
- Validação cliente: ≥ 0, número.
- ⌘↵ salva, esc cancela.
- Notas opcionais — se preenchidas, viram parte do `details` no audit
  (`"...; nota: Avaliação anual"`).

### 4. API binding

```typescript
api.updateAssetPrice(id: string, price: string, note?: string)
  → Promise<{price, price_updated_at, price_source}>
```

### 5. Estados

- Loading: botão "Salvar" desabilitado + spinner.
- Sucesso: fecha modal, atualiza KPI no AssetDetail, toast verde.
- Erro 422 (não MANUAL): toast "Este ativo tem fonte automática.
  Use 'Atualizar preço'." (raro — botão correto já deveria estar
  visível, mas defesa).
- Erro genérico: toast com `reason`.

## Out of Scope

- Edição de preço bulk pra vários assets MANUAL de uma vez.
- Histórico de preços manuais (audit log já registra cada edit).
- Schedule "lembre-me de atualizar daqui a 90 dias" — V2.

## Acceptance Criteria

1. `PATCH /api/assets/{id}/price` aceita preço para asset MANUAL e
   retorna 200 com novos valores.
2. Retorna 422 com mensagem clara quando asset não é MANUAL.
3. Audit log tem entrada `price.update.manual` com before/after no
   `details`.
4. Modal abre clicando "Editar preço" no AssetDetail.
5. Modal valida número ≥ 0 antes de submit.
6. Após salvar, AssetDetail mostra novo preço + dot verde (FRESH).
7. Notes do modal aparecem no audit details quando preenchidas.
8. `pytest -q` verde (≥3 testes: ok, 422 não-manual, audit).
