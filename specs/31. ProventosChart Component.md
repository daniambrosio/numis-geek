## Overview

Componente React reusável que renderiza o chart de proventos mensais
com 3 toggles (breakdown · currency · period), KPIs no header, e barras
empilhadas com hover. Consumido pela página `/distributions` (spec 32)
e pelo Dashboard (spec 33). Configurável via props para variar entre
"full" (página dedicada) e "compact" (card de dashboard).

## In Scope

### 1. Localização

`frontend/src/components/ProventosChart.tsx`

### 2. Props

```typescript
type Breakdown = "klass" | "country" | "fi" | "type" | "total";
type Currency = "BRL" | "USD";
type Period = "12m" | "24m" | "ytd";

interface ProventosChartProps {
  defaultBreakdown?: Breakdown;     // default "klass"
  defaultCurrency?: Currency;       // default "BRL"
  defaultPeriod?: Period;           // default "12m"
  includeSynthetic?: boolean;       // default true
  onIncludeSyntheticChange?: (v: boolean) => void;
  compact?: boolean;                // default false → esconde legenda + footer toggle
  hideToggles?: boolean;            // default false → esconde os 3 segmented controls
  noCard?: boolean;                 // default false → renderiza sem wrapper <Card>
}
```

### 3. Estado interno

const [breakdown, setBreakdown] = useState(defaultBreakdown ?? "klass");

- `compact={true}`: oculta legenda + footer toggle. Mantém chart + KPI line.

### 5. Sub-componente `<ProventosStackedBar />`

Recebe `data: ChartData` e renderiza SVG-free (divs flexbox) ou SVG
(decisão de implementação — protótipo usa divs com `bg-color`). Cada
barra:

- `rx=2` (cantos arredondados)
- opacity `0.85` default, `1` em hover
- outline thin no hover
- segmentos empilhados na ordem da legenda

Eixo X: labels de mês abreviados (`Mai`, `Jun`...). Se cols > 14:
saltar alternados.

Tooltip flutuante absoluto no canto superior direito do chart durante
hover, mostrando segmento por segmento com valor e %.

### 6. KPI computation

Os 3 KPIs vêm do response (`totals` + última row + penúltima):

```typescript
const lastMonth = data.rows[data.rows.length - 1];
const prevMonth = data.rows[data.rows.length - 2];
const mom = prevMonth ? (lastMonth.total - prevMonth.total) / prevMonth.total : null;
```

- "Último mês": `lastMonth.total` formatado
- "MoM": `mom` com sinal e cor
- "12M trailing" / "Média/mês" / "YTD": `data.totals.sum / monthly_avg / max`
  (backend já calcula)

### 7. Formatação

`fmtChart(value, currency, compact)`:
- BRL: `R$ 5.231` (sem decimais) ou `R$ 5,2 mil` (compact)
- USD: `US$ 1,2k` (compact) ou `US$ 1.231` (full)

Reusar `frontend/src/lib/format.ts` se existir; senão criar.

### 8. Cores do design system

Backend manda `color` em cada segment. Frontend respeita. Para
`breakdown=type`, as cores são fixas (§2.3 do delta doc); para outros
breakdowns, backend escolhe da paleta.

## Out of Scope

- Drill-down ao clicar numa barra (Open item)
- Animações de entrada (V1.5 se houver tempo)
- Export PNG/CSV
- Filtros dentro do chart (por FI, asset, etc.)

## Acceptance Criteria

1. Componente renderiza com defaults sem erros.
2. Trocar qualquer toggle re-fetcha (visível no devtools network).
3. Hover em barra mostra tooltip com segmentos + %.
4. `compact + hideToggles + noCard` rende uma versão minúscula sem
   cromo (usado pelo Dashboard).
5. `includeSynthetic` controlado externamente (prop) atualiza chart
   sem trocar state interno.
6. Build limpo (TS strict).
