# Spec 17a — Dashboard Re-skin (partial, current-backend)

## Status: 🚧 In Progress

## Overview

Third spec of **Fase A**. Replaces the placeholder Dashboard with a real one
that matches the prototype's visual language and renders the data we can
compute today against the current backend. Cards that depend on future
specs (08 / 09 / 10 / 11) ship as visible placeholders with a one-line hint
pointing at the unblocking spec, so the dashboard shows its full intended
layout from day one.

**No backend changes.**

---

## What we can compute today

For each Asset, the existing `api.getAssetPosition` returns
`total_invested_brl`, `total_received_brl`, `quantity_held`, `average_cost`.
We fan out these calls on mount (181 in parallel, browser caps at 6
concurrent — finishes in 1–2s for local dev) and aggregate client-side.

Derived figures:
- **Patrimônio investido (BRL)** = Σ `total_invested_brl`
- **Total recebido (BRL)** = Σ `total_received_brl` (mostly 0 today — DIVIDENDO/JUROS/JCP rows are empty; will jump after spec 07c data is re-imported / spec 08 lands)
- **Alocação por classe** = group `total_invested_brl` by `collapsedOf(asset.asset_class)`
- **Distribuição por custodiante** = group `total_invested_brl` by `financial_institution_id`
- **Atividade recente** = last 10 Lançamentos via `api.listLancamentos({ page_size: 10 })`

## In Scope

### Hero card (top)

Two-column layout:
- **Left:** "Patrimônio investido" KPI in BRL (big number with `.money tnum`), helper line "Patrimônio total ativa com specs 10/11 (Caixa + Cartões)". Three smaller pills below: Investimentos / Caixa / Cartões abertos — Investimentos has the real number, Caixa and Cartões show "—" with tooltip pointing at the spec.
- **Right:** 12-month evolution sparkline — **placeholder block** with "spec futura" hint (snapshot data doesn't exist yet).

### Row 1 — three side-by-side cards

- **Alocação por classe** — DonutChart on the left (~184px), legend on the right (top 5 classes + "Outros"), each row: dot · label · `tnum` % · `tnum money` BRL compact.
- **BR vs US** — placeholder Card with hint "ativa com spec 09 (Asset.country)".
- **Proventos recebidos · 12 meses** — placeholder Card. Show total `total_received_brl` (sum across assets) in a single KPI. Bar chart of monthly distribution is the placeholder.

### Row 2 — two side-by-side cards

- **Distribuição por custodiante** — top 5 FIs with FILogo + HBar + value + %. "Outros (N custodiantes)" line aggregates the rest.
- **Top movers** — placeholder Card with hint "ativa com spec 09 (current_price)".

### Row 3 — full-width

- **Atividade recente** — last 10 Lançamentos grouped by date, with a colored class strip, ticker/name, type badge, currency-pilled amount. Click → `/lancamentos` (no detail panel from here in this spec).

### Primitives

Add to `components/ui.tsx`:
- `SectionTitle` — small uppercase heading, with optional `action` slot on right
- `Trend` — small inline `+X.X%` / `−X.X%` green/red pill (no usage in this spec, but worth defining)

Add `components/charts.tsx` with:
- `DonutChart({ data, size, stroke })` — pure SVG, mirrors prototype
- `HBar({ value, max, color, height })` — pure SVG

`Sparkline` and `BarMonths` from the prototype are NOT ported in this spec
— the cards that would use them are placeholders, so the components can
arrive when the data does (spec 09 history / spec 08 distributions).

## Out of Scope

- Anything that requires `Asset.country`, `Asset.current_price`, `Transaction`, `CreditCard`, `Distribution`, `PTAXRate` — those cards stay placeholder
- USD totals (no PTAX yet) — show BRL only in the hero
- 12-month historical snapshots (no `position_snapshot` table — explicitly deferred per spec roadmap)
- Top movers (needs price data)
- Real activity feed mixing AssetMovement + Distribution + Transaction — only Lançamentos shown
- shadcn/ui — bespoke components
- recharts / visx — pure SVG via the new `charts.tsx`
