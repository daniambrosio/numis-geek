## Overview

`/snapshots/{ym}` tinha 2 limitações que escondiam dados úteis:

1. Tabela "Posições congeladas" cortava em 20 ativos (hardcoded). User
   tem ~32 ativos ativos → 12 invisíveis sem trabalho extra.
2. Card "Proventos do mês" só mostrava agregação por tipo. O KPI
   "N eventos" não tinha onde clicar — você sabia que tinha N
   proventos mas não conseguia ver quais.

Spec 45 entrega as duas correções num único PR frontend-only.

## In Scope

### 1. Positions — remover slice
- `frontend/src/pages/SnapshotDetail.tsx` linha 728: `sortedPositions.slice(0, 20).map(...)` → `sortedPositions.map(...)`.
- Wrapper `<div className="overflow-x-auto -mx-1">` ganha `max-h-[600px] overflow-y-auto` quando count > 20 (sane viewport limit).
- Rodapé "+ N ativos" sai (não é mais necessário).
- Header continua mostrando count: "Posições congeladas · N ativos".

### 2. Card "Eventos do mês" (NEW)
- Full-width Card inserido após o grid composição+proventos-por-tipo,
  antes dos Top movers.
- Header: "Eventos do mês · N proventos".
- Tabela com 6 colunas: Data (dd/mm), Ativo (ticker mono + nome),
  Tipo (dot colorido + label via `TYPE_META`), Bruto, IRRF (ou "—"
  se 0/null), Líquido (emerald-700).
- Ordenação por `event_date` ASC (cronológica).
- Renderização condicional: só aparece quando
  `distributions.length > 0`.
- Reusa `fmtBRL` (já existia) + novo `fmtMoney(n, currency)` helper
  local (multi-currency).

### 3. Tests
- `frontend/src/pages/SnapshotDetail.test.tsx` (novo) — 3 testes
  com mocks de api.*:
  1. Renderiza > 20 linhas na tabela de posições.
  2. Renderiza tabela "Eventos do mês" quando há proventos.
  3. Esconde tabela "Eventos do mês" quando não há proventos.

## Out of Scope

- **Backend** — zero mudança. `listSnapshotItems` já retorna todos os
  items; `listDistributions` já tem `page_size: 200` (folgado pra 1 mês).
- **Export CSV** — botão continua disabled (`Em breve`). Vira spec própria.
- **Filtro/search** na tabela de posições — premature. Adicionar quando
  o portfólio ultrapassar 100 ativos.
- **Virtualização** — tabelas até 500 rows funcionam sem virtualização.
- **Tabbed view** (Resumo / Eventos) — deliberada: os dois são
  complementares e leves o suficiente pra coexistir scrollados.
