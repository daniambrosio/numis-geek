# 00. Foundation

## Purpose
Set up the Python project scaffolding, database layer, migration system, and test infrastructure that all other features will build on.

## Tech Stack

| Concern | Choice | Rationale |
|---|---|---|
| Language | Python 3.12+ | Per project spec |
| ORM | SQLAlchemy 2.x | Database-agnostic; swap SQLite в†’ PostgreSQL on VPS without rewriting models |
| Migrations | Alembic | Standard companion to SQLAlchemy |
| Testing | pytest | Standard Python test runner |
| Password hashing | passlib[bcrypt] | Used by User & Workspace feature |
| Config | python-dotenv | `.env` file for local config (DB path, secret key, etc.) |

## Database Session Pattern (`db/session.py`)

- `get_engine()` вЂ” reads `DATABASE_URL` from config; defaults to `sqlite:///./numis_geek.db`
- `SessionLocal` вЂ” `sessionmaker` bound to the engine
- `get_db()` вЂ” context manager / generator for dependency injection

## Base Model (`db/base.py`)

- Single `Base = DeclarativeBase()` shared across all models
- All models import from here so Alembic autogenerate sees every table

## Alembic Configuration

- `alembic/env.py` imports `Base.metadata` from `numis_geek.db.base`
- `DATABASE_URL` read from environment (same as app config)
- Autogenerate enabled (`target_metadata = Base.metadata`)

## Test Infrastructure (`tests/conftest.py`)

```python
# In-memory SQLite for tests вЂ” fast, isolated, no files left behind
engine = create_engine("sqlite:///:memory:")
Base.metadata.create_all(engine)
Session = sessionmaker(bind=engine)

@pytest.fixture
def db():
    session = Session()
    yield session
    session.rollback()
    session.close()
```
