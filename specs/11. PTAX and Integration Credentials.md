# Spec 11 — PTAX (BCB) + Integration Credentials infra

## Status: 🚧 In Progress

## Overview

Hoje `services/positions.py` (linhas 159–168) trata só BRL: pra qualquer
ativo USD ele deixa `current_value_brl = None`. Isso bloqueia
"Patrimônio Investido" real no dashboard, snapshot mensal honesto, e
qualquer KPI MoM/YoY.

Spec 11 corrige isso na raiz com 3 entregáveis acoplados:

1. **PTAX diária USD/BRL** persistida (`ptax_rate` table) com ingest
   oficial via BCB SGS (séries 10813 venda + 10812 compra, gratuito e
   sem auth).
2. **FX service** `fx_rate_on(date) → Decimal` com walkback automático
   pra dia útil anterior (fim de semana/feriado).
3. **Integration credentials infra** — tabela + sysadmin UI pra
   gerenciar tokens de provider (brapi, Finnhub, etc.) sem mexer em
   código. Specs 12+ reusam.

E liga o `positions.py` à conversão real USD→BRL via `fx_rate_on`.

## In Scope

### Backend models
- `models/ptax_rate.py` — uma linha por data útil (date UNIQUE, buy
  Numeric(18,8), sell Numeric(18,8), source, fetched_at)
- `models/integration_credential.py` — provider enum (BCB, BRAPI,
  FINNHUB, YFINANCE), key_name, secret_value (plaintext),
  last_tested_at, last_test_result, last_test_message; workspace_id
  nullable (sempre NULL nesta spec; UI/API só lida com system-wide)

### Alembic migration
- `e4f5a6b7c8d9_ptax_and_integrations.py` — cria 2 tabelas + índices +
  unique constraints. Sem data ops (criação pura), sem
  `batch_alter_table`.

### Integrations layer
- `src/numis_geek/integrations/bcb.py` — `fetch_ptax_range(start, end)`
  via httpx, faz 2 chamadas (sell + buy), junta por data, retorna
  lista de PTAXRow

### Services
- `services/fx.py` — `fx_rate_on(db, date, *, side='sell', max_walkback_days=10) → Decimal`,
  exceção `FxRateNotFound`
- `services/ptax_sync.py` — `sync_ptax(db, *, mode='incremental'|'full')
  → PtaxSyncResult` (idempotente; mode full deriva janela do
  earliest AssetMovement.event_date com fallback 5y)
- `services/positions.py` (modify lines 159–168) — usar `fx_rate_on`
  pra USD assets, fallback silencioso se rate não encontrada

### API routes (sysadmin-only)
- `routes/integrations.py`:
  - GET /sysadmin/integrations — lista mascarada (•••• + last 4)
  - POST /sysadmin/integrations — cria credencial
  - PATCH /sysadmin/integrations/{id} — atualiza
  - DELETE /sysadmin/integrations/{id} — soft-delete
  - POST /sysadmin/integrations/{id}/test — ping provider, grava resultado
- `routes/ptax.py`:
  - GET /sysadmin/ptax — paginado desc por date
  - GET /sysadmin/ptax/status — last_date, total_rows, oldest_date, last_sync_at
  - POST /sysadmin/ptax/sync — body `{mode}` → PtaxSyncResult

### CLI script
- `scripts/import_ptax_from_bcb.py` — wrapper sobre `sync_ptax`
  service, args `--mode full|incremental`

### Frontend
- `lib/api.ts` — tipos `IntegrationProvider`, `IntegrationCredentialOut/Request`,
  `PTAXRateOut`, `PtaxSyncResult`, `PtaxStatusOut` + 6 helpers
- `pages/sysadmin/Integrations.tsx` — tabela + modal CRUD + botão Testar;

- `App.tsx` — rotas /sysadmin/integrations e /sysadmin/ptax
- `components/AppLayout.tsx` — 2 entradas novas na seção Estrutura

### Tests
