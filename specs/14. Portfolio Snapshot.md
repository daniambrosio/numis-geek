# Spec 14 — PortfolioSnapshot

## Status: ✅ Done (sem backfill Notion — fica como follow-up)

## Overview

Captura fotografia mensal da carteira (workspace × period_end_date). Usa
`current_price` do momento + PTAX da data + posições calculadas pelo
positions service. Estrutura preparada pro dashboard v2 (MoM/YoY) e pro
backfill de Notion.

## In Scope

- Models `PortfolioSnapshot` (header) + `PortfolioSnapshotItem` (1 por asset).
- Migration `a6b7c8d9e0f1` cria as 2 tabelas.
- Service `create_snapshot(db, workspace_id, period_end)` —
  itera assets ativos do workspace, computa position com `as_of=period_end`,
  guarda quantity, unit_price, market_value_native/brl/usd, total_invested_brl.
- Service `list_snapshots(workspace_id)`.
- CLI `python -m scripts.create_snapshot --workspace "X" --period 2026-04-30`.
- API `/snapshots` (GET list, POST create, GET items).
- 3 testes: cria snapshot, replace_if_exists, skipa qty=0.

## Out of Scope (vai pra follow-up)

- **Backfill do Notion** (DB IG Lote/Apuracao → snapshots históricos + relatório de
  divergências) — depende de yfinance + brapi histórico pra preços de fechamento.
  Plano em memory `corporate_actions_design`. Faço numa próxima sessão
  conectado ao MCP do Notion + 1-a-1 com o user.
- Snapshot agendado (cron). Por enquanto manual via CLI/UI.
- UI dedicada (vem na spec 15 Dashboard v2 + página de Patrimônio).
