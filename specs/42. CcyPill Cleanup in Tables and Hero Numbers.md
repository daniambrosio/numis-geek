## Source of truth

- Sessão de design 2026-05-26 (parte 4): user reportou que a pill BRL
  abaixo de cada valor da coluna NET na página `/lançamentos` polui
  visualmente — `fmtMoney` já prefixa com `R$`/`$`, então a pill é
  dupla sinalização.
- Spec 41 — Dual-currency Hero Views, §4 "Regra nasce sem pill":
  estabeleceu o princípio de que CcyPill só vive em valores
  decontextualizados. Esta spec **executa** esse princípio nas telas
  pré-existentes.
- `prototypes/index.html` (auditoria de 23 usos de `<CcyPill>`):
  - 5 tabelas/hero a limpar (alvos desta spec, listados em §1)
  - 12+ detail panels / KpiTile / asset header — onde a pill
    **fica** (decontextualizada)

## Overview

Spec 41 entregou dual-currency em hero views e estabeleceu a regra
"`<CcyPill>` só onde o valor está decontextualizado". Esta spec
**aplica a regra retroativamente** ao app existente: remove pills
redundantes em 5 lugares específicos sem mexer em estrutura nem
comportamento.

**Filosofia**: zero feature nova, zero refator de componente, zero
endpoint novo. É **limpeza visual** — reduzir ruído sem perder
informação. Cada remoção é justificada por "o prefixo `R$`/`$` ou
par converso já comunica a moeda".

**Onde a pill sobrevive (12+ lugares, NÃO MEXER)**:
- Detail panels (Lançamento, Provento, AssetDetail, ContaDetail) —
  valor flutua isolado num campo "Moeda" sem coluna explicando
- KpiTile com `ccy` prop (genérico, usado em vários lugares)
- Tabela de Movimentações: linha USD ganha pill como **marcador
  de exceção** (BRL é maioria implícita; USD é a anomalia)
- Cards de allocation BRL/USD na Dashboard (legenda semântica)

## In Scope

### Alvo 1 — Lançamentos table, coluna NET

`frontend/src/pages/AssetMovements.tsx` (ou nome equivalente).

Hoje (proto `~5081`):
```tsx
<td className="px-2 text-right">
  <div className={`tnum money font-medium ${...}`}>{fmtMoney(m.net, m.ccy, { sign: true })}</div>
  <CcyPill ccy={m.ccy}/>
</td>
```

Vira:
```tsx
<td className="px-2 text-right">
  <div className={`tnum money font-medium ${...}`}>{fmtMoney(m.net, m.ccy, { sign: true })}</div>
</td>
```

Justificativa: `fmtMoney` já prefixa com `R$` ou `$`. O usuário vê
"R$ −3.890,00" ou "−$2,091.00". A pill abaixo é redundante.

### Regra documentada — atualizar design-system doc

Criar (ou estender) `docs/design-system.md` com seção
"`<CcyPill>` — when to use":

```markdown
## `<CcyPill ccy="BRL|USD"/>`

### Use quando
- O valor flutua isolado, sem coluna explicando a moeda
  (ex.: campo "Moeda" num detail panel)
- A pill marca uma **exceção** numa lista onde a maioria é
  implícita (ex.: USD numa lista majoritariamente BRL onde
  o prefixo `R$` está omitido)

### NÃO use quando
- O valor já tem prefixo de símbolo visível (`R$ 3.890,00`,
  `$ 7.76`) — duplica sinalização
- Existe sub-linha com par converso (`R$ X / $ Y`) — o
  stacking já é o disambiguador
- A coluna da tabela já especifica a moeda no cabeçalho
- O valor é uma porcentagem ou outro número adimensional
```

Adicionar também o snippet canônico do dual-display da Spec 41
pra centralizar padrões monetários.

## Out of Scope

- **Refator do componente `<CcyPill>`** em si. Continua existindo,
  só usado em menos lugares.
- **Toggle global de moeda** (continua fora; Spec future quando
  user pedir).
- **Conversão em qualquer tabela** (Spec 41 já fechou: tabelas
  densas ficam na ccy original).
- **Pills em detail panels** — esses ficam intocados.
- **Pill na linha USD de Movimentações** (proto `~3746`) —
  permanece como marcador de exceção (BRL é maioria implícita).
- **CcyPill no list item de detail row** (proto `~2915`) — caso
  específico de Activity feed do Dashboard onde a pill é
  semanticamente útil; reabrir se virar pedido.
