# Spec 10 — Asset.financial_institution_id → Asset.account_id

## Status: 🚧 In Progress

## Overview

Highest data-risk spec in the roadmap. Three coupled changes on Asset's
ownership chain:

1. **Delete 5 redundant `CASH` "Saldo em Conta (X)" stubs.** Per memory
   `cash-stubs-are-redundant`, those rows are zero-lançamento metadata
   artifacts; broker float lives on the InvestmentAccount itself.
2. **Create 2 implicit investment accounts** for FIs that hold assets but
   have no investment account yet (`Particular → "Patrimônio pessoal"`,
   `Caixa → "Conta FGTS Caixa"`).
3. **Refactor FK chain:** drop `Asset.financial_institution_id`, add
   `Asset.account_id` (NOT NULL, FK → Account). The FI is now reachable
   only via `asset.account.financial_institution_id`.

Post-migration counts:
- asset: 181 → 176 (5 CASH stubs deleted)
- account: 13 → 15 (2 implicit accounts created)
- Every asset has a non-null account_id pointing at an `investment` account
  in the same workspace + FI as its previous `financial_institution_id`.

---

## Backfill rules

For each asset, find the investment account at the SAME (workspace, FI)
and link `asset.account_id` to it. The 2 implicit accounts cover the
previously-orphan FIs:

| FI | Account name | Currency | Opening balance | Used by |
|---|---|---|---|---|
| Particular | "Patrimônio pessoal" | BRL | 0 | 4 assets (Toyota, Volvo, Casa, Terreno) |
| Caixa | "Conta FGTS Caixa" | BRL | 0 | 3 assets (FGTS Carrefour/Eletrobras/Meli) |

Wise's 1 orphan asset (`Saldo em Conta (Wise)`) is a CASH stub deleted in
step 1, so Wise needs no implicit account.

## In Scope

### Backend
- `models/asset.py`:
  - Drop `financial_institution_id` column
  - Add `account_id: str` FK to `account.id`, NOT NULL
- Alembic migration `d3e4f5a6b7c8`:
  - DELETE 5 CASH stubs (named "Saldo em Conta%" + class=CASH)
  - INSERT 2 implicit investment accounts (Particular + Caixa)
  - Add `asset.account_id` (nullable initially)
  - Backfill: each asset's account_id = the investment account at
    its (workspace_id, financial_institution_id)
  - Verify zero null after backfill, then ALTER COLUMN NOT NULL
  - Drop `asset.financial_institution_id`
- `api/routes/assets.py`:
  - Drop `financial_institution_id` from AssetRequest
  - Add `account_id` to AssetRequest (required)
  - `AssetOut.financial_institution_id` / `_name` are derived from
    `asset.account.financial_institution_id` for backwards compat
  - Validate the supplied account is investment type
- `services/positions.py`: no change — still walks lançamentos directly
- `scripts/import_notion_assets.py`:
  - Resolve account from (workspace, FI short name + asset class) —
    pick the workspace's investment account at that FI; fall back to
    the implicit "Particular → Patrimônio pessoal" or "Caixa → Conta
    FGTS Caixa" automatically

### Frontend
- `lib/api.ts`: drop `financial_institution_id` from `AssetRequest`,
  add `account_id`. Keep `financial_institution_id` + `_name` on
  `AssetOut` for now (computed server-side from `asset.account.fi`).
- `AssetModal`: replace FI selector with Account selector (only
  investment accounts shown). The FI is shown read-only derived from
  the picked account.
- `AssetTable` / `AssetDetailPanel`: no change visible (still uses
  `financial_institution_name` from AssetOut).
- `pages/Dashboard.tsx`: no change — custodian distribution still
  works (FI derived server-side).

### Tests
- `test_assets.py`: payloads switch `financial_institution_id` →
  `account_id`. Update seed fixtures to create at least one investment
  account per FI.
- `test_asset_movements.py`, `test_positions.py`,
  `test_distributions.py`, `test_notion_import.py`: same Asset()
  ORM-construction updates — pass `account_id` instead of
  `financial_institution_id`.
