## Overview

Página de patrimônio investimento com:
1. **Hero** — total investido em BRL/USD + sparkline 12 meses.
2. **Donut por classe** + breakdown lista.
3. **Donut por país** (BR vs US) + barras.
4. **Por custodiante** (FI logo + barra + qtd ativos).
5. **Histórico 12m** (barras empilhadas por classe).
6. **Top 10 posições** (tabela clicável → AssetDetail).

Backend agrega tudo num único endpoint `GET /portfolio`. Frontend renderiza
seções.

### Estratégia de fonte de dados (importante)

Dois caminhos pra "current value":

- **Live**: `Asset.current_price` × posição (`services/positions.py`).
  Precisa de `current_price` populado (rodar refresh de preços).
- **Snapshot**: último `PortfolioSnapshot` + `PortfolioSnapshotItem`
  (já existem 28 snapshots backfilados do Notion).

Lógica:
- Se houver `PortfolioSnapshot` no workspace, usa o mais recente como
  `as_of`. Garante consistência com a história 12m (que vem desses
  mesmos snapshots).
- Cada item já carrega `market_value_brl`/`market_value_usd` — usar
  direto.
- "Live mode" entra depois quando current_price for confiável; por ora
  o endpoint sempre serve com `source = "snapshot"`.

## In Scope

### 1. Endpoint `GET /portfolio`

Query params: `workspace_id?` (sysadmin cross-workspace).

Response shape:

```python
class PortfolioOut(BaseModel):
    as_of: str                          # ISO date (snapshot period_end)
    source: str                         # "snapshot" | "live"
    ptax_rate: float | None             # BRL per USD at as_of
    total_value_brl: float
    total_value_usd: float
    total_invested_brl: float
    by_class: list[ClassBreakdown]      # [{class, label, color, value_brl, pct}]
    by_country: list[CountryBreakdown]  # [{country, value_brl, pct}]
    by_custodian: list[CustodianBreakdown]
        # [{fi_id, fi_short, fi_logo_slug, value_brl, pct, asset_count}]
    top_holdings: list[HoldingOut]      # 10 maiores
    history: list[HistoryPoint]
        # [{period_end, total_brl, by_class: {STOCK: X, REIT: Y, ...}}]
```

### 2. Service `services/portfolio_summary.py`

Single function `compute_portfolio_summary(db, workspace_id, *, sysadmin=False)`:
1. Fetch latest snapshot for the workspace.
2. Load all items + join with Asset → AssetClass, country, account → FI.
3. Aggregate:
   - by_class: sum market_value_brl grouped by asset_class.
   - by_country: sum grouped by asset.country.
   - by_custodian: sum grouped by account.financial_institution_id.
   - top_holdings: order by market_value_brl desc, limit 10.
4. History: last 12 snapshots ordered ASC, with per-class breakdown.

If no snapshot exists: return empty payload with `source = "live"` and
fallback to live positions (current_price + qty).

## Out of Scope

- "Live" mode (depends on current_price refresh routinely running).
- Patrimônio TOTAL (investments + cash − cards). This page mostra só
  **investido**, conforme prototype. Net worth completo entra com o
  Dashboard quando Transactions existirem.
- Drilldown views (clicar numa classe filtra outra seção).
- Allocation target overlay (entra no Spec 24 Decision-support).

## Acceptance Criteria

1. `GET /portfolio` retorna 200 com dados do último snapshot quando
   existe ≥1 snapshot no workspace.
2. Retorna 12 pontos no histórico (ou todos os snapshots se < 12).
3. Top 10 ordenado por value_brl desc.
4. Frontend `/portfolio` renderiza sem erros, mostra todas as seções.
5. Click numa linha de top holdings navega pra `/assets/:id`.
6. Click num custodiante (placeholder por enquanto — Spec 22 wire).
7. pytest verde (tests para portfolio summary service).
8. npm build limpo.
