# Spec 30 — `GET /distributions/chart` Endpoint

## Status: ✅ Done — 2026-05-24

## Source of truth

- `docs/prototype-deltas-2026-05-23.md` §2.1, §2.2
- Spec 29 (`aggregate_proventos`)

## Overview

Endpoint HTTP que expõe a função `aggregate_proventos` do Spec 29. Ele é
**puramente leitura**, retorna data pronta pra render, sem lógica nova.

## In Scope

### 1. Rota

```text
GET /api/distributions/chart
  query:
    workspace_id?     UUID (sysadmin only)
    period            "12m" | "24m" | "ytd"             (default "12m")
    breakdown         "klass" | "country" | "fi" | "type" | "total"
                                                       (default "klass")
    currency          "BRL" | "USD"                    (default "BRL")
    include_synthetic bool                             (default true)
  resp 200: ChartDataOut
```

### 2. Schema response

```python
class ChartSegmentOut(BaseModel):
    key: str
    label: str
    color: str
    value: Decimal | None = None  # None na legenda

class ChartRowOut(BaseModel):
    ym: str
    total: Decimal
    segments: list[ChartSegmentOut]

class ChartTotalsOut(BaseModel):
    sum: Decimal
    monthly_avg: Decimal
    max: Decimal

class ChartDataOut(BaseModel):
    rows: list[ChartRowOut]
    legend: list[ChartSegmentOut]
    totals: ChartTotalsOut
    currency: Literal["BRL", "USD"]
```

### 3. Handler

`api/routes/distributions.py` (existe — adicionar route):

```python
@router.get("/chart", response_model=ChartDataOut)
def get_distributions_chart(
    period: Period = "12m",
    breakdown: Breakdown = "klass",
    currency: Currency = "BRL",
    include_synthetic: bool = True,
    workspace_id: str | None = None,
    user: User = Depends(current_user),
    db: Session = Depends(get_db),
):
    ws = resolve_workspace(user, workspace_id)
    data = aggregate_proventos(db, ws, period=period, breakdown=breakdown,
                                currency=currency, include_synthetic=include_synthetic)
    return data
```

Sysadmin pode passar `workspace_id`; outros usam o próprio.

### 4. Frontend API binding

`frontend/src/lib/api.ts`:

```typescript
export interface ChartSegment { key: string; label: string; color: string; value?: string }
export interface ChartRow    { ym: string; total: string; segments: ChartSegment[] }
export interface ChartTotals { sum: string; monthly_avg: string; max: string }
export interface ChartData   { rows: ChartRow[]; legend: ChartSegment[]; totals: ChartTotals; currency: "BRL"|"USD" }

api.getDistributionsChart(params: {
  period?: "12m"|"24m"|"ytd",
  breakdown?: "klass"|"country"|"fi"|"type"|"total",
  currency?: "BRL"|"USD",
  include_synthetic?: boolean,
}): Promise<ChartData>
```

### 5. Validation

- Enum validation via FastAPI (`Literal`).
- Período inválido → 422.
- `breakdown` inválido → 422.
- Workspace não existe / sem acesso → 403.

## Out of Scope

- Caching server-side. Latência esperada < 200ms para 24m de dados;
  reavaliar se ficar > 500ms.
- Filtros adicionais (por asset, FI específico, type específico) — Spec 21
  já tem filtros na lista; chart é deliberadamente menos granular.
- Drill-down (clicar numa barra → lista do mês).

## Acceptance Criteria

1. `GET /api/distributions/chart` retorna 200 com defaults.
2. Trocar `period=24m` retorna mais buckets.
3. Trocar `currency=USD` retorna totais convertidos.
4. `include_synthetic=false` esconde rows OPTION_PREMIUM mas mantém
   legenda com a key quando `breakdown=type`.
5. Sysadmin com `workspace_id` cross-workspace funciona.
6. Member tentando `workspace_id` de outro workspace → 403.
7. `pytest -q` verde (≥6 testes endpoint).
