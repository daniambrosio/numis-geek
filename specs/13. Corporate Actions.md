# Spec 13 — CorporateAction (Split / Grouping / Conversion)

## Status: ✅ Done

## Overview

Adiciona suporte pra eventos de transformação de posição (split,
agrupamento, conversão de ativo). Entidade separada de `AssetMovement` —
não são trades, são transformações que preservam o custo total. Ver
memory `corporate_actions_design`.

## In Scope

- Model `CorporateAction` (workspace_id, asset_id, event_date, event_type,
  ratio, target_asset_id?, target_ratio?, notes, is_active)
- Migration `f5a6b7c8d9e0`
- positions.py merge cronológico de AssetMovements + CorporateActions.
  SPLIT/GROUPING escalam qty e basis_qty pelo ratio (basis_cost preservado);
  ASSET_CONVERSION fecha a posição (target tem suas próprias movements).
- CRUD `/corporate-actions` (list/create/delete soft) com workspace guard
- 4 testes cobrindo split, grouping, inactive ignored, asset conversion

## Mechanics (positions walk)

| Evento | qty | avg_cost | total_cost |
|---|---|---|---|
| SPLIT ratio=R | × R | / R | preservado |
| GROUPING ratio=R<1 | × R | / R | preservado |
| ASSET_CONVERSION | → 0 | reset | reset |

- Backfill automático do Notion (vem na spec 14)
