# 02. User Management and Profile

## Purpose
Give admins a UI to manage workspace users and give every user a profile page to view their data, change their name, and change their password. Also establishes the shared app layout (topbar + collapsible sidebar) and the dark/light theme system.

## Backend Changes

### Model update: `user.name`
Add optional `name` field to the `user` table.

| Column | Type | Notes |
|---|---|---|
| name | str(255) \| NULL | Display name, editable by the user |

### New endpoints (`/users`)

| Method | Route | Access | Description |
|---|---|---|---|
| GET | /users | admin | List all users in the workspace |
| POST | /users/invite | admin | Create a new user |
| PUT | /users/{id}/role | admin | Change a user's role |
| PUT | /users/{id}/deactivate | admin | Deactivate a user |
| GET | /users/me | any | Get current user's profile |
| PUT | /users/me | any | Update current user's name |
| PUT | /users/me/password | any | Change current user's password |

### Access enforcement
- Admin-only routes return 403 if called by a member
- `/users/me/password` validates current password before saving

## Frontend

### AppLayout (`src/components/AppLayout.tsx`)
Shared layout used by all authenticated pages.

```
┌──────────────────────────────────────────┐
│ TOPBAR: [≡] Logo        [☀/🌙] [avatar▾]│
├──────────┬───────────────────────────────┤
│ SIDEBAR  │  {children}                   │
│ (colaps) │                               │
└──────────┴───────────────────────────────┘
```

- Toggle button (≡) collapses/expands sidebar; state persisted in localStorage
- Theme toggle (sun/moon icon) in topbar, always visible
- Avatar dropdown (top right): "Meu Perfil" and "Sair"
- Sidebar links: Dashboard, (admin only) Usuários
- Admin-only links hidden from members

### Theme system (`src/lib/theme.ts`)
- Stores preference in localStorage (`theme`: `"dark"` | `"light"`)
- Applies/removes `dark` class on `<html>` element
- `useTheme()` hook exposes `{ theme, toggleTheme }`
- Defaults to system preference on first visit

### Profile page (`/profile`)
- Read-only section: email, name, role, member since
- Edit name form (single field, inline save)
- Change password form: current password + new password + confirm

### User Management page (`/admin/users`) — admin only
- Table: name, email, role badge, status (active/inactive), actions
- "Convidar Usuário" button → modal with: name, email, initial password, role
- Per-row actions: change role (dropdown), deactivate (button with confirmation)
