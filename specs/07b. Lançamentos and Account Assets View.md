# Spec 07b — Lançamentos (Entries) + Account Assets View

## Status: 📝 Planned

## Overview

This spec ships the **Lançamentos** (entries) feature — every event that moves an asset's quantity, value, or pays cash from it: purchases, sales, dividends, interest, come-cotas, stock dividends, rights issues, JCP. Lançamentos are the heart of the investment domain; without them, Asset is just a static catalog.

It also ships a small UX add-on: a new **"Ativos"** tab inside the Contas page that groups assets by custodian — making it easy to answer "what do I hold at XP?" without leaving the Contas context.

This is the "07b" half of the 07a→07b→07c sequence: 07a populated real Asset data from Notion; 07b adds the Lançamento model and UI; 07c will import historical Lançamentos from Notion.

---

## Non-goals (explicitly deferred)

- **Aporte / resgate** (cash transfers between accounts) — separate spec; needs Account-side cash flow modeling. Not in MVP types.
- **Position snapshot table** (`position_snapshot`) — current position is **computed on-the-fly** from lançamentos in v1. Materialization is an optimization for later if performance demands it.
- **Realized P&L, FIFO/LIFO basis tracking** — calculated in views, not stored.
- **Recurring lançamentos** (e.g., monthly dividend schedule) — manual entry only in v1.
- **Reconciliation against statement files / trade notes** — separate spec.
- **PTAX table** — only `fx_rate` field on Lançamento now (manual or `1.0`); the dedicated PTAX spec backfills later.
- **Notion Lançamento import** — Spec 07c.
- **External_id / external_source** columns — deferred to 07c, where the question will be re-asked with more detail.

## Lançamento types

Enum `LancamentoType` (English values, Portuguese display in UI):

| Enum value | Display (PT) | When |
|---|---|---|
| `COMPRA` | Compra | Buying an asset |
| `VENDA` | Venda | Selling an asset |
| `DIVIDENDO` | Dividendo | Cash dividend received |
| `JUROS` | Juros / Cupom | Interest or coupon payment (FIXED_INCOME, BOND, FGTS) |
| `JCP` | JCP | Juros sobre capital próprio (BR stocks) |
| `COME_COTAS` | Come-cotas | Semi-annual forced tax event (FUND, PRIVATE_PENSION) |
| `BONIFICACAO` | Bonificação | Stock dividend (free new shares) |
| `SUBSCRICAO` | Subscrição | Rights subscription |

## Model: `lancamento`

Single table with optional columns; validation per `type` enforced in the API layer.

| Column | Type | Notes |
|---|---|---|
| id | UUID PK | |
| workspace_id | UUID FK → workspace | NOT NULL |
| asset_id | UUID FK → asset | NOT NULL — the asset this entry relates to |
| type | enum `LancamentoType` | NOT NULL |
| event_date | date | NOT NULL — the date of the event (compra, pagamento, etc.) |
| settlement_date | date \| None | optional — D+N for stocks/bonds when relevant |
| **Quantity / price (COMPRA, VENDA, BONIFICACAO, SUBSCRICAO)** | | |
| quantity | numeric(18,8) \| None | required for COMPRA/VENDA/BONIFICACAO/SUBSCRICAO |
| unit_price | numeric(18,8) \| None | required for COMPRA/VENDA/SUBSCRICAO; null for BONIFICACAO (free shares) |
| **Cash amounts (all types)** | | |
| gross_amount | numeric(18,2) \| None | computed default = quantity × unit_price; can be overridden for fixed-income / bonds where face_value × % matters |
| fee | numeric(18,2) \| None | brokerage / exchange fees |
| tax | numeric(18,2) \| None | IR retido na fonte, IOF, etc. |
| net_amount | numeric(18,2) | computed: gross_amount − fee − tax (for COMPRA: + fee + tax; for VENDA/income: − fee − tax) |
| **Currency** | | |
| currency | enum `Currency` (`BRL`/`USD`) | NOT NULL — currency of `gross_amount`/`fee`/`tax`/`net_amount`; usually equals `asset.currency` but can differ for FX events |
| fx_rate | numeric(18,8) | NOT NULL, default `1.0`; rate to convert `currency` ↔ workspace base currency (BRL). For BRL→BRL = 1; for USD→BRL = the day's PTAX. Hybrid hook for the future PTAX spec. |
| **Notes / metadata** | | |
| notes | text \| None | free-text |
| is_active | bool | default `True`; soft-delete (so we can "void" a wrong entry without losing audit trail) |
| created_at, updated_at | datetime | UTC auto |
| created_by, updated_by | UUID FK → user | |

**Indexes**
- `(workspace_id, event_date)` — for time-range queries
- `(asset_id, event_date)` — for per-asset history (the most common query)
- `(workspace_id, type, event_date)` — for filtered timelines (e.g., "all dividendos in 2026")

**No unique constraint** — duplicate lançamentos are allowed (e.g., two purchases on the same day at slightly different prices).

## Validation rules (per type)

| Field | COMPRA | VENDA | DIVIDENDO | JUROS | JCP | COME_COTAS | BONIFICACAO | SUBSCRICAO |
|---|---|---|---|---|---|---|---|---|
| `quantity` | req | req | — | — | — | — | req | req |
| `unit_price` | req | req | — | — | — | — | — | req |
| `gross_amount` | computed/override | computed/override | req | req | req | req | computed (= 0) | computed/override |
| `fee` | optional | optional | optional | optional | optional | — | — | optional |
| `tax` | optional | optional | optional | optional | optional | req | — | optional |

- `event_date` always required, must be ≤ today.
- For income types (DIVIDENDO/JUROS/JCP/COME_COTAS): `gross_amount` must be > 0 (use VENDA for negative cash effect, not negative dividendo).
- For BONIFICACAO: `unit_price` must be null and `gross_amount` defaults to 0 (free shares).
- For COME_COTAS: `quantity` is null (it doesn't change holdings, just collects tax).

## Position calculation (computed)

A view-layer helper computes per-asset position from lançamentos:

```
quantity_held(asset) = Σ (qty of COMPRA + BONIFICACAO + SUBSCRICAO) − Σ (qty of VENDA)
average_cost_brl(asset) = weighted average of (qty × unit_price × fx_rate) over COMPRA + SUBSCRICAO
                          (BONIFICACAO does not affect basis since unit_price is 0)
total_invested_brl(asset) = quantity_held × average_cost_brl
total_received_brl(asset) = Σ (net_amount × fx_rate of DIVIDENDO/JUROS/JCP) for that asset
```

These are functions/queries, not stored columns. Implemented in `src/numis_geek/services/positions.py`.

## API

### `/lancamentos`

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/lancamentos` | authenticated | List in caller's workspace; filters: `?asset_id=`, `?type=`, `?from=YYYY-MM-DD`, `?to=YYYY-MM-DD`, `?include_inactive=true`. Default sort: `event_date DESC, created_at DESC`. Paginated (default 50/page, max 200) |
| GET | `/lancamentos/{id}` | authenticated | Single lançamento |
| POST | `/lancamentos` | authenticated | Create. Workspace inferred from caller; sysadmin must pass `workspace_id` |
| PUT | `/lancamentos/{id}` | authenticated | Update |
| PUT | `/lancamentos/{id}/deactivate` | authenticated | Soft-delete |

Audit actions: `lancamento.created`, `lancamento.updated`, `lancamento.deactivated`.

### `/assets/{id}/positions` (read-only computed)

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/assets/{id}/position` | authenticated | Returns current position: `{ quantity_held, average_cost, average_cost_brl, total_invested_brl, total_received_brl, currency }` |
| GET | `/assets/{id}/lancamentos` | authenticated | Convenience: list all lançamentos for the asset (same shape as `/lancamentos?asset_id=`) |

### `/accounts/by-custodian` (for the Ativos tab)

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/accounts/by-custodian` | authenticated | Returns `[{ financial_institution: {id, short_name, logo_slug}, accounts: [...investment accounts at this FI...], assets: [...assets at this FI...] }]`. Skips FIs with zero investment accounts AND zero assets. Sysadmin gets cross-workspace via `?workspace_id=`. |

## Frontend

### `/lancamentos` page

- Route `/lancamentos`
- Page file: `frontend/src/pages/Lancamentos.tsx`
- Table columns: event_date, type (badge), asset (ticker / name link), quantity, unit_price, net_amount, currency, actions
- Filters: asset (autocomplete), type (multi-select), date range, "show inactive"
- Create / edit modal:
  - Type selector first → reveals appropriate fields per the validation table above
  - Asset autocomplete → defaults `currency` from asset
  - `fx_rate` field with default `1.0` and a small hint "Cotação do dia (PTAX automático futuramente)"
- Sidebar: "Lançamentos" item under the workspace section

### `/assets/:id` (small detail panel addition)

When viewing an asset (link from `/assets`), show:
- The current position (qty held, avg cost, total invested, total received)
- A timeline of recent lançamentos for that asset
- "+ Novo Lançamento" button pre-filling the asset

### `/accounts` page — "Ativos" tab (the user's "visão dos ativos dentro da conta")

- The existing `/accounts` page gets two tabs at the top: **Contas** (current) | **Ativos**
- The "Ativos" tab calls `GET /accounts/by-custodian` and renders, for each custodian:
  - Header row: FI logo / name + summary count ("3 ativos · 1 conta investimento")
  - Account row(s) under that FI (investment-type only)
  - Asset list under that FI (clickable → go to asset detail / `/assets?fi=X`)
  - Sections collapsible by default; expanded if only 1 FI
- Empty state: "Nenhum ativo cadastrado neste custodian" if FI has accounts but no assets, etc.
- Skips checking-account-only FIs in this tab.

### Sidebar additions

- "Lançamentos" (workspace-scoped) appears in the main nav for all authenticated users.
