# 01. User and Workspace

## Purpose
Establish the top-level workspace concept and user authentication/authorization system. Every other entity in the system belongs to a workspace.

## Decisions

| Decision | Choice |
|---|---|
| Auth method | Email + password (bcrypt via passlib) |
| Workspace model | Single shared workspace per installation; users are invited by an admin |
| Roles | `admin` (full access) and `member` (view + enter transactions/entries) |
| Session | JWT token returned on login (stateless, ready for VPS/API use) |

## Auth Flow

### Login
- `POST /auth/login` â€” body: `{ email, password }`
- Normalize email to lowercase before lookup
- Verify password with `passlib.verify`
- On success: return signed JWT (`{ user_id, workspace_id, role, exp }`)
- On failure: return 401 (same message for unknown email and wrong password â€” no enumeration)

### Logout
- Client discards the token (stateless JWT â€” no server-side invalidation in v1)

### Token validation
- All protected routes validate the JWT and inject the current user context
- Expired or invalid tokens â†’ 401
