# Spec 16 — Frontend Shell Rebuild

## Status: 🚧 In Progress

## Overview

First spec of the **Fase A — Frontend re-skin** track. Rebuilds the
application shell (`AppLayout`) to match the prototype in
`prototypes/index.html`: bifurcated sidebar with section headers, redesigned
top bar with new toggles (privacy, comfort), CSS tokens for asset-class
colors and currency pills, and a `.money` class for one-toggle privacy
blurring.

**No backend changes.** Existing pages keep working unchanged inside the new
shell — they will be re-skinned individually in specs 17a-17d. New sidebar
entries that don't have pages yet route to a generic `<ComingSoon />`
placeholder.

### CSS tokens (added to `frontend/src/index.css`)

- `.money` — base class on every monetary span. When `body.privacy` is set, applies `filter: blur(6px)` and hover unblurs.
- `.tnum` — `font-variant-numeric: tabular-nums`. Default-applied via the existing `body` font stack? No — opt-in via class on numeric columns/spans.
- `body.comfort` — scales `text-[10px]`, `text-[11px]`, `text-xs` (`12px`), `text-[13px]` up by 1–2px each. Implemented as `body.comfort .text-xs { font-size: 13px }` etc.

### TypeScript token maps (new `frontend/src/lib/tokens.ts`)

- `KLASS_COLORS: Record<AssetClass, { bg, text, border, dot }>` — 11 classes per `docs/conceptual-model.md` §2.4. Returns Tailwind class strings keyed by class code. Used later by Ativos/dashboard re-skins; **defined here, not consumed yet**.
- `CURRENCY_PILL: Record<'BRL'|'USD', { bg, text }>` — BRL amber, USD emerald.

Note: the 11 classes are the **target** schema. Backend still uses 14 classes today; until spec 09 collapses them, the frontend token map will include mappings for the legacy codes (`STOCK_BR`, `STOCK_US`, `FII`, `BOND`) pointing to the same color as their merge target (`STOCK`, `REIT`, `FIXED_INCOME`). One-line `// TODO(spec-09): remove legacy aliases`.

### Privacy / Comfort modules

- New `frontend/src/lib/privacy.ts` — `getPrivacy(): boolean`, `setPrivacy(b)`, `togglePrivacy()`. Persists `privacy` flag in `localStorage`. Applies `body.privacy` class.
- New `frontend/src/lib/comfort.ts` — same shape.

Both initialize on app load (similar pattern to `theme.ts`).

### Placeholder page

New `frontend/src/components/ComingSoon.tsx` — single component that takes a `title` prop and renders a centered card with the page name and a one-line "Em breve" note. Used by all 7 new routes.
