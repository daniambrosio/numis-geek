# Spec 54 — Build Version Display + Mismatch Detection

## Status: 🚧 Implementing

## Why

Hoje a app não expõe versão. O sidebar mostra um `v1` hardcoded
(`AppLayout.tsx:186`); o `pyproject.toml` está em `0.1.0` desde o
commit zero; o `package.json` em `0.0.0`. FastAPI declara
`version="0.1.0"` no `app` mas isso não muda também.

Problemas concretos:

1. **Não dá pra saber qual versão tá em produção.** Quando algo dá
   ruim, o usuário tem que perguntar "qual commit?" — o sistema
   deveria mostrar.
2. **Frontend cacheado vs backend novo = bug silencioso.** Após
   deploy, o browser pode estar segurando bundles antigos por horas;
   um endpoint API que mudou shape devolve 422 sem explicação. Sem
   detecção, o usuário só vê erros aleatórios.
3. **Sem semver, sem changelog útil.** Tags são `spec-NN` mas as 17+
   últimas specs nem foram tagged. Não tem rastreio de "o que mudou
   entre vX e vY".

## Decisões locked (entrevista 2026-06-04)

| # | Pergunta | Resposta |
|---|---|---|
| 1 | Formato da versão exibida | **Semver + sha + data**: `0.52.0 · 6bd9dc1 · 2026-06-04` |
| 2 | Como bumpar | **Manual via `scripts/bump.sh patch\|minor\|major`** — edita pyproject.toml + package.json + cria tag git. |
| 3 | Onde mostrar no UI | **Footer fixo no sidebar esquerdo** (sempre visível). Hover/tooltip mostra full info. |
| 4 | Mismatch backend↔frontend | **Banner "Nova versão, atualize"** com botão de hard reload. |

## In Scope

### 1. Source of truth

- `pyproject.toml.version` — semver autoritativo.
- `frontend/package.json.version` — espelha o pyproject (sincronizado
  por `bump.sh`).
- `GIT_SHA` + `BUILD_DATE` — variáveis de ambiente injetadas pelo
  Docker build (via `--build-arg` no `deploy.sh`).
- **Dev mode** (sem env vars): backend deriva sha via
  `git rev-parse --short HEAD` e data via `date.today()`. Frontend
  via plugin do Vite com mesmas fallbacks.

### 2. Backend — `/api/version` endpoint

`src/numis_geek/version.py` (módulo novo):

```python
def get_version_info() -> dict:
    return {
        "version": _semver(),   # de pyproject via importlib.metadata
        "sha":     _sha(),      # GIT_SHA env ou `git rev-parse --short`
        "date":    _build_date(),  # BUILD_DATE env ou today
    }
```

`src/numis_geek/api/app.py`:
- `app = FastAPI(..., version=_semver())` (dinâmico).
- `GET /version` (público, sem auth) retorna `get_version_info()`.

### 3. Frontend — bake version no bundle

`frontend/vite.config.ts`:

```typescript
import { execSync } from 'node:child_process'
import pkg from './package.json' assert { type: 'json' }

const gitSha = process.env.GIT_SHA
  ?? execSync('git rev-parse --short HEAD').toString().trim()
const buildDate = process.env.BUILD_DATE
  ?? new Date().toISOString().slice(0, 10)

export default defineConfig({
  define: {
    __APP_VERSION__: JSON.stringify(pkg.version),
    __APP_SHA__:     JSON.stringify(gitSha),
    __APP_BUILD_DATE__: JSON.stringify(buildDate),
  },
  // ...
})
```

`frontend/src/vite-env.d.ts` declara os 3 globals como `string`.

### 5. Frontend — mismatch detection + banner

Componente novo `frontend/src/components/VersionMismatchBanner.tsx`:

- Componente renderizado em `AppLayout` acima do `<header>` do main.
- Hook: faz `GET /api/version` na montagem + a cada 5 min.
- Compara `data.sha !== __APP_SHA__`. Se diferente, mostra:

  ```
  ⚠ Nova versão disponível (0.53.0 · ab12cd3) — [Atualizar]
  ```

- Botão "Atualizar" faz `window.location.reload(true)` (hard reload).
- Banner fica colapsável (X pra dismissar até a próxima polling tick).

`frontend/src/lib/api.ts`: adiciona `api.version()` typed retornando
`{ version: string; sha: string; date: string }`.

### 6. Bump script — `scripts/bump.sh`

```bash
#!/usr/bin/env bash
# Usage: scripts/bump.sh patch|minor|major
set -euo pipefail
PART=${1:?patch|minor|major}

# Read current
CUR=$(grep -m1 '^version = ' pyproject.toml | cut -d'"' -f2)
IFS='.' read -r MAJ MIN PAT <<< "$CUR"
case "$PART" in
  major) MAJ=$((MAJ+1)); MIN=0; PAT=0 ;;
  minor) MIN=$((MIN+1)); PAT=0 ;;
  patch) PAT=$((PAT+1)) ;;
  *) echo "Use patch|minor|major"; exit 1 ;;
esac
NEW="$MAJ.$MIN.$PAT"

# Write back
sed -i.bak "s/^version = .*/version = \"$NEW\"/" pyproject.toml && rm pyproject.toml.bak
node -e "const p=require('./frontend/package.json'); p.version='$NEW'; require('fs').writeFileSync('frontend/package.json', JSON.stringify(p,null,2)+'\n')"

git add pyproject.toml frontend/package.json
echo "Bumped $CUR → $NEW. Commit, then run: git tag v$NEW"
```

Voltado pra rodar **antes** do commit final de uma spec/release. Não
cria tag automaticamente (você decide quando taggear).

### 7. Build wiring

`Dockerfile`:
- ARG `GIT_SHA`, ARG `BUILD_DATE`.
- Passar como ENV pro runtime do backend.
- Passar como env vars pro `npm run build` no stage frontend.

`deploy.sh`:
- Antes do `docker build`, calcular `GIT_SHA=$(git rev-parse --short HEAD)`
  e `BUILD_DATE=$(date +%F)`.
- Passar via `docker build --build-arg GIT_SHA=$GIT_SHA --build-arg BUILD_DATE=$BUILD_DATE`.

### 9. Smoke check

1. `npm run build` no frontend; verifica que o sha aparece embeded no
   JS gerado (`grep` no bundle).
2. Inicia backend; `curl localhost:8000/version` retorna 3 chaves.
3. Abre app no browser; ve a versão no rodapé do sidebar.
4. Bump backend sem rebuild do frontend; abre app cacheado; banner
   aparece em até 5 min.

## Out of Scope

- Conventional Commits automation — bump é manual.
- Changelog gen (CHANGELOG.md). Spec separada se quiser.
- Tag automation (`git tag v$NEW` automático). Voce decide quando.
- Frontend SW (service worker) com auto-update — banner manual é
  suficiente pra dev/prod-pequena.
- Backend autoreboot quando detecta mismatch frontend. Sem sentido.
