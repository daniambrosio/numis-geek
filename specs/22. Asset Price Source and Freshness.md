## Overview

Sobe o sistema de preços de "tem ou não tem" pra "tem, de onde veio, e
quão fresco está". Introduz `PriceSource` como enum tipado no `Asset` e
define os thresholds de staleness que toda UI consultará. Este spec é
**só schema + types + helper service** — endpoints e UI ficam nos
specs 23–28.

### Por que separar

A coluna `price_source` é leitura crítica em qualquer endpoint que
liste assets (popover do topbar, página Ativos, AssetDetail) e em todo
write de preço (refresh API + PATCH manual). Acoplar isso ao spec do
endpoint (23) atrasaria UI; melhor pôr schema na primeira leva e deixar
o resto incremental.

## In Scope

### 1. Enum `PriceSource`

Em `src/numis_geek/models/asset.py`:

class PriceSource(str, enum.Enum):

    TESOURO = "TESOURO"    # Tesouro Direto
    MANUAL = "MANUAL"      # Imóvel, veículo, FGTS, VGBL, CDB, LCI, etc.

> protótipo sugere. yfinance saiu da estratégia

### 2. Nova coluna em `Asset`

```python
price_source: Mapped[PriceSource | None] = mapped_column(
    Enum(PriceSource), nullable=True
)
```

Nullable porque ativos importados antes deste spec não têm fonte
declarada. O backfill da migration (abaixo) preenche a maioria; o
resto vira `MANUAL`.

### 3. Migration `alembic/versions/xxxx_asset_price_source.py`

1. `op.add_column("asset", price_source ENUM(...) NULL)` —
   usar `op.batch_alter_table` (SQLite).
2. Backfill executado por `scripts/backfill_asset_price_source.py`
   (separado da migration; dry-run + `--apply` pattern):
   - `CRYPTO` → `COINBASE`
   - `STOCK/REIT/ETF` + `country=BR` → `BRAPI`
   - `STOCK/REIT/ETF` + `country=US` → `FINNHUB`
   - Resto (`REAL_ESTATE`, `VEHICLE`, `FGTS`, `PRIVATE_PENSION`,
     `CASH`, `FUND`, `OPTION`, `FIXED_INCOME` inclusive Tesouro Direto)
     → `MANUAL`
3. Snapshot pattern: `cp numis_geek.db numis_geek.db.bak-before-22`
   antes; verificar contagem pré/pós por source ([[migration_snapshot_pattern]]).

### Decisão TESOURO (2026-05-24)

`TESOURO` permanece no enum mas **não há adapter no V1**. Os 21
títulos Tesouro Direto foram backfilados como `MANUAL`. Spec 23 não
inclui adapter de Tesouro. User pode editar `price_source` per-asset
no futuro se decidir construir o adapter.

### Decisão OPTION (2026-05-24, pós-sync)

Após o primeiro refresh manual, brapi falhou nos 2 OPTION ativos
existentes (cobertura esparsa pra opções B3). Decisão: OPTIONs vão pra
`MANUAL` por padrão; sem adapter dedicado no V1. Os 2 já em `BRAPI`
foram migrados pra `MANUAL` por UPDATE direto.

### 4. Freshness tiers

Sem persistência — só helpers em `src/numis_geek/services/price_freshness.py`:

```python
class PriceTier(str, enum.Enum):
    FRESH = "FRESH"     # verde
    STALE = "STALE"     # âmbar
    OLD = "OLD"         # vermelho
    UNKNOWN = "UNKNOWN" # nunca atualizado (price_updated_at IS NULL)

def freshness_tier(
    updated_at: datetime | None,
    source: PriceSource | None,
    *, now: datetime | None = None,
) -> PriceTier:
    """
    APIs (BRAPI, FINNHUB, COINBASE, TESOURO):
        fresh < 24h · stale < 7d · old >= 7d
    Manual:
        fresh < 30d · stale < 90d · old >= 90d
    """
```
