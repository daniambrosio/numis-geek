# Spec 08 — AssetMovement rename + Distribution split + drop Asset.subtype

## Status: 🚧 In Progress

## Overview

First spec of **Fase B** (backend refactor). Three coupled changes that
land together because they all touch `Lancamento` rows and the income-type
enum values:

1. **Rename** `Lancamento` → `AssetMovement`. Tabela `lancamento` →
   `asset_movement`. Enum values renamed to English (COMPRA→BUY,
   VENDA→SELL, BONIFICACAO→BONUS, SUBSCRICAO→SUBSCRIPTION,
   RESGATE_TOTAL→FULL_REDEMPTION, COME_COTAS mantém).
2. **Split** `DIVIDENDO`, `JUROS`, `JCP` out of the type enum into a new
   `Distribution` entity. New fourth type `SECURITIES_LENDING` ("Aluguel" /
   "BTC"). `asset_id` is nullable on Distribution (Avenue's generic
   "rendimento de aluguel").
3. **Drop** `Asset.subtype`. Non-null subtype values are preserved by
   appending to `Asset.notes` with prefix `[ex-subtype]` before the
   column is dropped.

**Data shape today (snapshot):** 181 assets, 660 lançamentos, 75 assets
with non-null subtype. Zero rows in DIVIDENDO/JUROS/JCP — Distribution
starts empty.

Each step compiles/tests in isolation so we can stop and roll back at any
clean step boundary.
