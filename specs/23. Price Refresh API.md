# Spec 23 — Price Refresh API + per-source adapters

## Overview

Generaliza o serviço de refresh existente (Spec 12) para:
- aceitar filtro por **fonte** ou por **lista de asset_ids**;
- expor um endpoint único `POST /api/prices/refresh` (substitui o
  `POST /assets/refresh-prices/bulk` em forma; o antigo pode redirecionar
  ou ser deprecado);
- adicionar adapters novos para `COINBASE` (cripto) e `TESOURO`
  (Tesouro Direto);
- gravar evento de auditoria em todo refresh.

Endpoint individual `POST /assets/{id}/refresh-price` permanece com
shape minimamente ajustado (retorna `price_source` e bloqueia MANUAL
com 422).

## In Scope

### 1. Endpoint generalizado

```text
POST /api/prices/refresh
  body:
    {
      "source": "BRAPI" | "FINNHUB" | "COINBASE" | "TESOURO" | null,
      "asset_ids": ["uuid", ...] | null
    }
  rules:
    - null body / both null  → refresh em todos os assets com source != MANUAL
    - source != null         → refresh só dessa fonte
    - asset_ids != null      → refresh só desses (MANUAL é skipped + reportado)
    - source + asset_ids     → AND
  resp 200:
    {
      "ok":     <int>,
      "failed": <int>,
      "skipped": <int>,           # MANUAL ou sem ticker
      "errors": [
        { "asset_id": "uuid", "ticker": "PETR4", "reason": "timeout" },
        ...
      ],
      "ran_at": "2026-05-24T18:00:01-03:00"
    }
```

Auth: `member`+. Sysadmin pode passar `workspace_id` query param.

### 2. Endpoint individual (já existe — ajustar)

```text
POST /api/assets/{id}/refresh-price
  resp 200:
    {
      "price": "39.12",
      "price_updated_at": "2026-05-24T18:00:01-03:00",
      "price_source": "BRAPI"
    }
  resp 422 se asset.price_source == MANUAL
```

### 3. Endpoint deprecado

`POST /assets/refresh-prices/bulk` (Spec 12) — manter por compat por
agora mas redirecionar internamente pro novo handler (sem mudar
contrato). Adicionar deprecation header `Sunset: <date>`. Remoção em
spec futuro.

### 4. Adapters

Em `src/numis_geek/integrations/`:

| Source | Arquivo | Função | Auth |
|---|---|---|---|
| `BRAPI` | `brapi.py` (existe) | `fetch_quote(ticker, token)` | token via `IntegrationCredential` |
| `FINNHUB` | `finnhub.py` (existe) | `fetch_quote(symbol, token)` | token |
| `COINBASE` | `coinbase.py` (**novo**) | `fetch_spot(symbol)` → ex.: `BTC-USD` | sem token |

Coinbase endpoint público: `https://api.coinbase.com/v2/prices/{pair}/spot`.

> **TESOURO deferido (decisão 2026-05-24, Spec 22):** o enum value
> `TESOURO` existe mas não há adapter no V1. Tesouro Direto assets
> ficaram como `MANUAL`. Se o user quiser auto-pricing, construir o
> adapter aqui em spec futuro.

### 5. Roteamento por source

`services/price_update.py` ganha:

```python
def refresh_by_source(db, source: PriceSource, workspace_id: str | None) -> RefreshSummary
def refresh_by_ids(db, asset_ids: list[str], workspace_id: str | None) -> RefreshSummary
def refresh_all_automated(db, workspace_id: str | None) -> RefreshSummary
```

Cada uma chama `_refresh_one(db, asset)` que despacha por
`asset.price_source` para o adapter correto, atualiza `current_price`
+ `price_updated_at`, e emite audit.

### 6. Audit

Cada asset refrescado com sucesso emite via `AuditService`:

```
action: "price.refresh"
resource: "asset/{id}"
details: "PETR4: 38,90 → 39,12 (BRAPI)"
```

Erros não geram audit (só entram no `errors[]` da resposta).

Falhas de adapter (timeout, 5xx, parse) viram `reason` no error item;
o handler **não levanta** — segue pro próximo asset.

### 7. Schemas Pydantic

```python
class RefreshRequest(BaseModel):
    source: PriceSource | None = None
    asset_ids: list[str] | None = None

class RefreshError(BaseModel):
    asset_id: str
    ticker: str | None
    reason: str

class RefreshSummary(BaseModel):
    ok: int
    failed: int
    skipped: int
    errors: list[RefreshError]
    ran_at: datetime
```

## Acceptance Criteria

1. `POST /api/prices/refresh` sem body atualiza todos os assets com
   source automatizada.
2. Com `source` filtra para aquela fonte só.
3. Com `asset_ids` filtra; passar id de asset MANUAL retorna no
   `skipped` (não falha o request).
4. `POST /api/assets/{id}/refresh-price` retorna 422 com mensagem
   clara quando MANUAL.
5. Coinbase adapter retorna preço de BTC/ETH em `USD`.
6. Tesouro adapter retorna preço unitário de um título com slug
   conhecido.
7. Audit log tem N entradas `price.refresh` após N refreshes
   bem-sucedidos.
8. `pytest -q` verde, incluindo mocks dos 4 adapters.
